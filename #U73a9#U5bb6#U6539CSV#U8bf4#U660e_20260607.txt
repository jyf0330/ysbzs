《元素背包史》CSV 改表说明

现在程序默认读取 data/csv 里的 CSV 文件。以后改数据，不用先改 JSON，不用改代码。

一、最常改的地方

1. 改我方初始宠物
文件：data/csv/10_初始阵容.csv
改：宠物ID、行(1-8)、列(1-8)、启用
效果：新建局后我方初始阵容变化。

2. 改敌方出怪
文件：data/csv/03_怪物波次_好读版.csv
改：天数、时段、回合、宠物ID、数量、位置
效果：对应天数/回合刷出的敌人变化。

3. 改宠物强度
文件：data/csv/01_宠物主表_好读版.csv
改：HP、攻、防、盾、行动、机制ID、标签等
效果：作为我方单位、商店商品、基础模板时生效。

4. 改怪物强度
文件：data/csv/02_怪物模板_好读版.csv
改：HP、攻、防、盾、行动、机制ID、机制参数
效果：敌方单位生成时使用这里的模板。

5. 改商店
文件：data/csv/06_商店奖励池_好读版.csv
改：解锁日、池档、默认价、价格覆盖、各类权重、商店池、奖励池
效果：商店刷新、奖励候选、价格和出现概率变化。

二、改完怎么验证

在项目根目录执行：

npm run check:csv
npm run check:all

如果只想看 CSV 数据有没有接上：

npm run data:check

如果想导出程序实际归一化后的 JSON 方便检查：

npm run data:dump

三、UI 怎么刷新新数据

如果服务没开：

npm run ui

如果 UI 服务已经开着：
- 点页面里的“新建局”；或
- 浏览器控制台执行：await __YSBZS__.command('NEW_GAME')；或
- 直接重启 npm run ui。

四、注意事项

1. 不要改 CSV 表头列名。
2. 宠物ID必须能在 01 宠物主表里找到。
3. 怪物波次里的宠物ID必须也能在 02 怪物模板里找到，否则校验会报错。
4. 多机制可以写英文逗号、中文逗号、顿号分隔，例如：mech_aura,mech_curse。
5. 旧机制ID会自动归一化，例如 mech_opening_shield → mech_shield_flat。
6. CSV 是当前真源；normalized_data.json 只是兜底，不再是优先真源。
