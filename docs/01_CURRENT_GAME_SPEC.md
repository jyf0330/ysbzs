# 01_CURRENT_GAME_SPEC · 游戏规格与核心规则

## 一句话定位

《元素背包史》是一个向 The Bazaar 对齐、节奏更快的**战术构筑游戏**。核心循环：战斗 → 商店 → 买英雄 → 同名合成升级 → 上阵组合 → 下一天。

采用 8×8 棋盘、元素叠层+引爆为主要战斗机制。第一阶段核心：**火伤害闭环 + 召唤流闭环**。

## 核心机制

### 元素系统

- 四种元素：风 → 火 → 水 → 土 → 风（循环克制）
- 后一元素克制前一元素（火克风、水克火、土克水、风克土）
- 元素叠层：同格同元素叠层，层数=N 时基础伤害=N，累计伤害=1+2+...+N
- 引爆阈值：同一元素在同一格叠到 ≥3 层时触发引爆
- 默认引爆范围：单体中心格（仅命中的那格），伤害作用于范围怪物
- 钻石火魔（fire_demon）在场上或背包/备战区时，火引爆范围扩展为十字 5 格（中心+上下左右），伤害作用于范围内怪物
- 怪物格上的元素不做十字爆炸，只做单体结算
- 战斗内零隐藏随机：所有伤害可预览

### 战斗流程

```
玩家回合（选择英雄→移动→选择行动槽→使用行动点）
  → 调用 settleExplosions() 结算元素引爆
  → 召唤物行动 runSummonActions()
  → 怪物回合 3AP
  → 小回合/阶段结束
```

### 召唤流闭环（第一阶段核心之一）

- 召唤物必须归属具体元素/英雄流派，**不做中立召唤**
- 召唤物自动攻击（继承 summoner 元素）
- 召唤物死亡留对应元素
- 治疗型/增益型召唤物辅助
- 火伤害闭环与召唤流闭环并列第一阶段目标

### Run 结构

- Run = 10 天
- 前 3 天：教学核心循环
- 第 4-10 天：成长、组合、压力与最终战
- 城堡 HP=80（代码实际值；旧文档 100 已废弃），跨战斗保留，不回满
- Day10 为最终战
- 胜利：敌方城堡 HP 归零
- 失败：我方城堡被毁或所有英雄倒下
- 英雄上阵：当前 2 个同时上阵，预留扩展至最多 4 个（后续版本）

### 商店规则

- **只卖英雄**（UNIT_DEFS 中的英雄卡），不卖行动槽、元素瓶、强化块、遗物
- 英雄档位四阶：青铜 → 白银 → 黄金 → 钻石（钻石为最高档位）
- 同名合成按档位升级，最终目标对齐青铜 → 白银 → 黄金 → 钻石
- 商店按 day_phase 池 + tier 过滤（SHOP_POOLS）

### 遗物系统

**已砍掉。** 遗物职能由英雄被动/主动承接。代码中若存在遗物残留（G.relics、relicBonus 等），需 TDD 清理。

## 设计支柱

1. 算得清才打得爽
2. 元素即一切
3. 连锁是高潮/召唤滚雪球是 v1 首验
4. 守城即节奏
5. 随机只在局外

## 反支柱

- 不做战斗内暴击/闪避/随机命中
- 不做背板隐藏机制
- 不做脱离元素/连锁的纯数值堆
