## 0. 本 GDD 的核心决策（必读）

> **不做独立路径地图。** roguelite 的「run 张力」由两件事提供：
> 1. **商店随机**（局外随机，已实现 `SHOP_POOLS`）。
> 2. **城堡 HP 跨战斗保留**（一条贯穿整局的血条）。
>
> **一天三阶段保留**，作为 run 的推进骨架：run = 多天 × （上午战 / 中午商店 / 下午战 / 夜晚商店）。
>
> **城堡是棋盘实体格 + HP**（代码现状），不抽象成纯血条。

这条把杀戮尖塔式「地图节点」替换为「多天阶段 + 跨战城堡 HP」，实现成本最低且复用现有代码。

---

## 1. 系统目的

把目前「单日 demo」升级为「**一整局 run**」：

- 多天连续推进，期间**城堡 HP 不回满**，形成「守住到通关」的张力。
- run 有明确**终点（Boss / 敌方城堡）**与**失败线（我方城堡破）**。
- 与 S1 召唤引擎、S2 成长/连锁配合：引擎在 run 中段成型，后段守城/打 Boss 兑现。

## 2. Run 结构

### 2.1 一局 = 5 天（可调）

| 天 | 上午 | 中午 | 下午 | 夜晚 | 备注 |
|---|---|---|---|---|---|
| Day1 | 战(2) | 商店 | 战(2) | 商店 | 教学 + 引擎入口（夜池召芽灵） |
| Day2 | 战(2) | 商店 | 战(3) | 商店 | 引擎成型期 |
| Day3 | 战(3) | 商店 | 战(3) | 商店 | 压力上升（厚皮/快速/冲锋） |
| Day4 | 战 | 商店 | 战 | 商店 | 精英登场 |
| Day5 | 战 | 商店 | **Boss** | — | 终局：打掉敌方城堡 / Boss |

- 小回合数沿用 `DAY_ROUND_CONFIG`（Day1–3 已配，Day4–5 待补）。
- 波次/怪种沿用 `DAY_WAVE_CONFIG`（已配到 Day5，Day5 含 boss）。
- **5 天为软上限**，可按 playtest 调整（3~7 天区间）。

### 2.2 阶段时序（保留现有）

```
DayStart → 上午战斗 → 中午商店 → 下午战斗 → 夜晚商店 → (次日) ... → Day5 Boss → 结算
```

> 与 `一天三阶段系统设计.md` 一致；该文档**不降级**，是 run 的推进骨架。

## 3. 城堡跨战系统

### 3.1 现状（代码已事实实现）

- `G.playerCastle{hp:100,maxHp:100,pos:(12,1)}`、`G.enemyCastle{hp:100,maxHp:100,pos:(0,11)}`。
- 怪物寻路可攻我方城堡（`damagePlayerCastle`）；元素十字波及可伤敌方城堡。
- **`closeShop`/`finishMonsters` 不重置城堡 HP** → 城堡 HP **已经跨天保留**。
- 胜负：`checkGameOver` 已含「敌方城堡毁=胜 / 我方城堡毁=败 / 全英雄死=败」。

> 即「跨战城堡 HP」无需新建系统，只需**确认不重置 + 调数值 + 补 Boss 日**。

### 3.2 数值（AI 定，标可调）

| 项 | 值 | 说明 |
|---|---|---|
| 我方城堡 HP | **100** | 整局 run 血条，**不自动回满** |
| 我方城堡夜间小回复 | **+0（默认关）** | 可选：夜晚商店 +5 回复作为难度旋钮 |
| 敌方城堡 HP | **100**（Day1–4）/ Boss 日另计 | 前期难以触及，引擎成型后才打得动 |
| 失败线 | 我方城堡 HP=0 任意时刻 | 立即 OVER |
| 胜利线 | 敌方城堡 HP=0 | 立即胜利（理论上可提前速通） |
| 软终局 | Day5 Boss | 撑到并击破即通关 |

### 3.3 设计意图

- **不回满**制造 roguelite 损耗：前期被磨城堡，逼玩家尽快让引擎成型来止血。
- **敌方城堡可提前拆**：给强构筑速通空间（呼应「引擎成型→大杀四方」）。
- **回复旋钮**留作难度层（S4 Ascension 可关掉回复 / 降初始 HP）。
