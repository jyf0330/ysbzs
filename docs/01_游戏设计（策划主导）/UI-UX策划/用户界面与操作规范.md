## 二、界面与模式

| 界面 / 模式 | 玩家可见入口 | 可执行操作 | 退出 / 切换方式 |
|---|---|---|---|
| 战斗界面 | 默认主界面 | 选择英雄、移动、选择行动槽、调整方向、使用行动点、查看元素、结束回合 | 点击结束回合或进入商店 / 游戏结束状态 |
| 棋盘元素 Tooltip | 点击元素格 | 查看元素叠层、查看是否将引爆、手动触发引爆 `[NEEDS_REVIEW]` | 点击引爆按钮或点击其他区域 `[NEEDS_REVIEW]` |
| 商店界面 | 波次结算后进入商店 | 查看商品、购买攻击方块 / 道具 / 元素强化 / 背包操作、完成购买 | 点击完成购买 |
| 行动点分配弹窗 | 从背包装备行动方块时触发 | 选择分配到哪个行动槽、取消 | 选择槽位或取消 |
| 消息弹窗 | 胜利 / 失败 / 提示类消息 | 阅读提示、关闭弹窗 | 点击弹窗按钮 |

## 三、输入动作表

| 操作意图 | 输入方式 | 生效条件 | 失败 / 禁用条件 | 反馈 |
|---|---|---|---|---|
| 选择英雄 | 点击英雄所在格 | 英雄存活且处于玩家可操作阶段 | 怪物行动 / 商店 / 游戏结束阶段 `[NEEDS_REVIEW]` | 英雄卡高亮，行动预览按当前英雄刷新 |
| 移动英雄 | 选中英雄后点击空格 | 目标格可达且未被阻挡 `[NEEDS_REVIEW]` | 非法格、被占用、阶段不允许 | 英雄位置更新，坐标与预览刷新 |
| 选择行动槽 | 点击行动槽卡片 | 行动槽未使用且处于玩家回合 | 已使用、无可用目标、阶段不允许 | 行动槽选中态高亮，棋盘显示攻击 / 元素预览 |
| 调整方向 | 点击方向按钮 | 行动槽支持方向且未使用 | 已使用或阶段不允许 | 方向按钮高亮，预览范围刷新 |
| 使用行动点 | 点击“使用”按钮 | 已选择合法行动槽和方向 | 行动槽已使用、目标不合法、阶段不允许 | 造成攻击 / 元素叠层预览，行动槽变为已使用 |
| 查看元素格 | 点击有元素的棋盘格 | 格子存在元素叠层 | 空格或无元素格 `[NEEDS_REVIEW]` | 打开 Tooltip，显示叠层与引爆状态 |
| 结束回合 | 点击“结束回合”按钮 | 玩家回合可结束 | 强制未完成操作 `[NEEDS_REVIEW]` | 进入怪物行动或商店 / 结算流程 |
| 购买商品 | 点击商品购买按钮 | 金币足够且商品可购买 | 金币不足、已购买、商店外阶段 | 金币扣除，商品状态更新 |
| 完成购买 | 点击“完成购买”按钮 | 商店阶段 | 非商店阶段 | 关闭商店，进入下一轮流程 |

## 四、控件行为

- 按钮默认态：可操作按钮使用高对比底色和明确文字。
- 按钮禁用态：不可购买 / 已使用 / 阶段不允许时应降低可见强调，并禁止触发核心动作。
- 选中态：英雄、行动槽、方向按钮需要有持续高亮，直到玩家切换选择或阶段结束。
- 危险操作确认：当前版本未定义二次确认；是否用于手动引爆、放弃回合等操作 `[NEEDS_REVIEW]`。
- 键盘 / 手柄 / 触屏输入：当前版本未定义 `[NEEDS_REVIEW]`。

## 五、反馈规则

- 操作成功：更新对应 HUD、棋盘预览、日志或弹窗。
- 操作失败：应说明不可操作原因，避免只是不响应 `[NEEDS_REVIEW]`。
- 资源不足：购买按钮禁用或点击后提示金币不足 `[NEEDS_REVIEW]`。
- 阶段不允许：按钮应禁用，棋盘点击应忽略或提示当前阶段 `[NEEDS_REVIEW]`。
- 模式切换：进入商店、怪物行动、游戏结束时，主操作区应同步隐藏或禁用不适用操作。

- 键盘快捷键尚未定义。

- 结束回合前是否需要未用行动点提醒仍需确认 `[NEEDS_REVIEW]`。
