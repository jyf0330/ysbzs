# 一天三阶段系统设计（设计概念草稿）

- 文档状态：设计概念草稿
- 当前阶段：规划中，尚未进入实现
- 目标方向：这是当前战斗验证台的下一个演化方向；在城堡系统落地之后引入

---

## 设计目标

1. 把当前的回合结构从简单的「玩家回合 → 怪物回合 → 商店回合」改为「早 → 午 → 晚 → 入夜」的一天循环。
2. 每个时段提供不同的可行动作和策略选择，增加决策深度。
3. 与城堡系统（保护己方城堡、击破敌方城堡）和 Super Auto Pets 式商店（最多 3 单位上阵）配合。

---

## 一天的结构

```
           玩家行动         怪物行动       结算 / 过渡
早上 (Morning)    玩家可操作 3 个行动槽  怪物 AI 回合     无
下午 (Noon)       玩家可操作 3 个行动槽  怪物 AI 回合     无
晚上 (Evening)    玩家可操作 3 个行动槽  怪物 AI 回合     入夜结算（生成下一波新怪）
```

### 每个时段内的子流程

```
┌─────────────────────────────────────────┐
│  时段开始                                │
│  → 刷新阶段相关效果/状态变化               │
│  → 玩家操作阶段（3 行动槽，可攻击/移动/道具）│
│  → 结束阶段（玩家点击"结束行动"）           │
│  → 怪物 AI 回合（当前场上怪物行动）          │
│  → 结算阶段（元素引爆/伤害/清除）            │
│  → 进入下一时段或入夜                      │
└─────────────────────────────────────────┘
```

## 时段规则

### 早上（Morning）

- 特点：开局阶段，恢复/增益效果触发的时机。
- 玩家行动：可操作 3 个行动槽，攻击棋盘上的怪物。
- 怪物行动：场上已有怪物正常执行 AI。
- 过渡：无特殊结算，直接进入下午。

### 下午（Noon）

- 特点：核心战斗时段，元素层可积累叠加。
- 玩家行动：再次操作 3 个行动槽。
- 怪物行动：场上怪物正常 AI。
- 过渡：无特殊结算。

### 晚上（Evening）

- 特点：战斗收尾，同时准备下一波的过渡阶段。
- 玩家行动：最后一次操作 3 个行动槽。
- 怪物行动：场上怪物正常 AI。
- 入夜结算：
  - 清除棋盘上残余元素层（或部分衰减）
  - 判断胜负条件（己方城堡 HP、敌方城堡 HP）
  - 如未结束，生成下一波怪物（从敌方城堡方向入场）
  - 进入商店阶段（Super Auto Pets 式购买）

### 商店阶段（在一天结束后）

- 按当前设计方向，商店改为 Super Auto Pets 式。
- 玩家从若干单位中选择购买，最多上阵 3 个单位。
- 单位自带攻击模块和基础属性（替换当前英雄+行动槽系统）。
- 商店后进入新一天的早上。

## 预期变化点

以下是当前战斗系统需要变化的方向（设计概念，尚未拆分为实现任务）：

| 当前系统 | 未来方向 |
|---|---|
| 固定 2 英雄，ha/hb | 最多 3 单位，玩家自选 |
| 英雄固定位置 | 单位可放置/移动/替换 |
| 6 个行动槽，每人 3 个 | 单位自带攻击模块，不再有独立行动槽 |
| 13×13 棋盘 | 10×10 棋盘，左下己方城堡，右上敌方城堡 |
| 金币商店随机攻击方块 | Super Auto Pets 式单位商店 |
| 无时间阶段 | 早/午/晚三天循环 |
| 无城堡 | 敌方城堡出怪，己方城堡受保护 |

## 暂未确定的问题

以下内容需要后续决策：

1. **时段数量**：目前定 3 个（早/午/晚）还是简化为 2 个（白天/黑夜）？
2. **行动槽与单位的关系**：每个单位每时段有固定行动次数，还是和当前一样有独立行动槽？
3. **入夜结算的具体规则**：元素层是否衰减？衰减比例？
4. **胜负条件**：己方城堡 HP 归零算输？还是也要达到特定目标？
5. **新怪生成位置**：从敌方城堡方向入场，具体机制（每时段出？每晚出？每 N 回合出？）
6. **与当前战斗验证台的关系**：当前验证台在城堡系统上线后是废弃，还是保留为训练关/教程关？
