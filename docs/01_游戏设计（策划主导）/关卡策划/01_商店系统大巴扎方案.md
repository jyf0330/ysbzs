# 大巴扎商店系统·借鉴与适配方案

> **版本**：v0.2
> **日期**：2026-05-28
> **状态**：借鉴分析 + 适配设计
> **前提**：本项目战斗系统为 ysbzs 自研回合战术引擎，不照搬 The Bazaar 的 auto-battle

---

## 一、总览：我们从 The Bazaar 借什么

The Bazaar 商店系统的核心创新，可以提炼为三个机制：

```
1. 多摊位标签过滤 —— 商品不是按类别分，而是按标签穿透多个摊位
2. 品级统一定价   —— 同品级同价，物品功能不影响价格
3. 角色即商品     —— 商店只卖"角色卡"，不卖零散装备/药水
```

我们全借，落地方式如下：

| 借什么 | The Bazaar 原生 | ysbzs 适配 |
|---|---|---|
| 多摊位 | 每位商人有固定商品池 | 每个摊位有标签白名单，动态过滤英雄池 |
| 标签系统 | 物品有 weapon/small/工具 等标签 | 每个英雄有 5-6 个标签（元素/位置/职责等） |
| 品级统一定价 | 青铜=2/白银=4/黄金=6/钻石=8 | 1阶=2/2阶=4/3阶=6 |
| 刷新成本递增 | 同一商人刷新费 1→2→3 递增 | 同一摊位刷新费首次免费→1→2→3 递增 |
| 冻结 | 冻结商品保留到下一轮 | 冻结摊位，该摊位商品不随刷新变化 |
| 角色即商品 | 只卖英雄/物品，不卖零散配件 | 只卖英雄（首版不做消耗品） |

## 二、多摊位标签过滤（主要借鉴）

### 2a. The Bazaar 的原生设计

在 The Bazaar 里，商店不是"按类别分区的超市"，而是多个商人各自卖特定标签的商品：

```
武器商人 → 只卖带 weapon tag 的物品
小型商人 → 只卖带 small tag 的物品
工具商人 → 只卖带 tool tag 的物品
```

一件物品因为有多个 tag（比如 "匕首" 同时有 weapon + small 两个 tag），
就会同时出现在武器商人和小型商人两个摊位。

### 2b. 我们怎么适配

我们把"物品 tag"换成"英雄标签"。每个英雄有 5-6 个标签，一个英雄可以出现在多个摊位：

```
摊位设定                        展示英雄
────────────────────────────────────────────────────
摊A: 标签白名单 = {火, 前排}    火苗使(火前排) + 岩盾使(土前排) + 召唤使(火中排)
                                 ↑ 这三个英雄都有"火"或"前排"标签

摊B: 标签白名单 = {水, 后排}    水滴使(水后排) + 洪流使(水范围)
                                 ↑ 这两个英雄标签同时包含"水"和"后排"

摊C: 标签白名单 = {远程}        水滴使(远程) + 微风使(远程) + 飓风使(远程)
```

**关键规则（来自 The Bazaar）：**
- 摊位展示**同时满足白名单全部标签**的英雄（AND 逻辑）
- 已拥有的英雄不再出现在摊位上（避免无意义的重复展示）
- 一个英雄因为标签多，出现在 **2-4 个摊位** → 玩家从不同入口都能找到同一个英雄

### 2c. 和 The Bazaar 的差异

| 维度 | The Bazaar | 我们的选择 | 原因 |
|---|---|---|---|
| 商品池来源 | 固定商人专属池 | 从全局英雄池按标签过滤 | 首版英雄数量少，固定池太薄 |
| 摊位数量 | 固定几个商人 | 随天数递增(2→3→4→5) | 匹配教学节奏，不一开始就用复杂度压玩家 |
| 标签数量/物品 | 1-3 个 | 5-6 个 | 英雄需要更多维度来区分，并且保证标签覆盖率 |
| 过滤逻辑 | 包含 tag | 全部标签匹配（AND）| 避免摊位商品过多，精准控制选择面 |

## 三、品级统一定价（主要借鉴）

### 3a. The Bazaar 的原生设计

The Bazaar 最大胆的设计之一：**不看物品功能，只看品级定价**。

```
青铜（Bronze）= 2 金币
白银（Silver）= 4 金币
黄金（Gold）  = 6 金币
钻石（Diamond）= 8 金币
```

一把 1 伤害的武器和一把 10 伤害的武器，如果是同品级，价格完全一样。
玩家买的是"有机会变强"，不是"花多少钱买多少战力"。

### 3b. 我们怎么适配

```
1阶（普通）= 2 金币  ← 对应 The Bazaar 的青铜
2阶（精英）= 4 金币  ← 对应 The Bazaar 的白银
3阶（传说）= 6 金币  ← 对应 The Bazaar 的黄金
```

**定价铁律（完整复制）：**
- ✅ 同品级同价，不看攻击形状大小
- ✅ 同品级同价，不看元素类型（火水风土一个价）
- ✅ 同品级同价，不看格子数（大十字和单点一个价）
- ✅ 只看品级，简单透明可预期

**但是要注意差异：**

The Bazaar 里同一商品可以通过升级（bronze → silver → gold → diamond）逐渐强化，价格也随之上升。我们对应的机制是：英雄同名**合成升阶**，每次升阶行动槽增强。

```
火苗使 1阶（2💰购入）
    → 再买一个火苗使 → 合成升到 2阶（槽1+HP↑）
    → 再买两个火苗使 → 合成升到 3阶（槽3→大十字）
```

### 3c. 品级与天数的关系（借鉴 The Bazaar 的节奏）

The Bazaar 的天数推进会让高品级物品出现概率递增，我们同样采用：

| 天数 | 摊位品级占比 | 对应 The Bazaar 的阶段 |
|---|---|---|
| 第1天 | 全 1阶 | 相当于开局的青铜过渡期 |
| 第2-3天 | 1阶70% + 2阶30% | 白银开始出现 |
| 第4-5天 | 1阶40% + 2阶50% + 3阶10% | 黄金入场 |
| 第6天+ | 1阶20% + 2阶50% + 3阶30% | 构筑成熟的终局阶段 |

## 四、刷新与冻结机制（直接借鉴）

### 4a. The Bazaar 的原生设计

The Bazaar 的刷新机制非常精妙：

- 每个商人的刷新费**独立计算**，和该商人对话第一次刷新最便宜
- 脱离对话再回来，刷新费**重置**
- 冻结可以锁定某个商品，但刷新后冻结解除
- 每次刷新**全部重置**（包括你刚看到的好货）

### 4b. 我们怎么适配

**摊位级别刷新（直接复制）：**

| 操作 | 费用 | 效果 |
|---|---|---|
| 第一次刷该摊位/夜 | **免费** | 该摊位商品全部重随 |
| 第二次刷该摊位/夜 | **1 金币** | |
| 第三次刷该摊位/夜 | **2 金币** | |
| 第四次及以上 | **3 金币** | |
| 切换天数 | 所有摊位刷新费**重置为免费** | |

**全局刷新（区别于 The Bazaar——The Bazaar 没有这个，是我们加的补充）：**

| 操作 | 费用 | 效果 |
|---|---|---|
| 全部摊位刷新 | **3 金币** | 所有摊位商品重随，冻结摊位除外 |

**冻结（简化为摊位级）：**

学习 The Bazaar 的"冻结档位保留"思路，但我们把冻结粒度从"单个商品"调整为"整个摊位"：

- ❄ 冻结该摊位 → 下次刷新时保留
- ❄ 冻结状态在下一天自动解除
- 冻结不消耗金币
- 冻结和刷新互斥：冻结的摊位不能被刷新

为什么做摊位级而不是商品级冻结？
- 首版摊位数量少（最多5个），摊位级冻结操作成本低
- 玩家只需要决定"这个方向我要保留"而不是"这3个商品我要逐个点冻结"
- 跟英雄标签系统匹配——保留摊位=保留那个流派方向

## 五、经济模型（借鉴+本地化）

### 5a. The Bazaar 的经济骨架

The Bazaar 的经济核心：

```
固定收入 + 利息 + PvP 胜场奖励
收入用于：买物品 + 刷新 + 升级
物品不可出售（有限例外）
```

### 5b. 我们怎么适配

```
固定收入（每日）+ 击杀怪物收入（替代 PvP 胜场）+ 利息
收入用于：买英雄 + 刷摊位 + 全局刷新
英雄可出售，但只返品级价-1 → 阻止无限买卖套利
```

| 项目 | The Bazaar | 我们的数值（首版） |
|---|---|---|
| 每日固定收入 | 取决于 PvP 表现 | Day1=3, Day2+=5 金币固定 |
| 额外收入 | PvP 胜场奖励 | 击杀怪物 1-2/只(上限5/天) |
| 利息 | 每 5 存款 +1，上限 +3 | 每 5 存款 +1，上限 +3（直接复制）|
| 出售返还 | 通常不可出售 | 品级价-1（1阶返1, 2阶返3, 3阶返5）|
| 刷新成本递增 | 同商人对话中递增 | 同摊位同一天内递增（直接复制）|
| 冻结 | 免费，商品级 | 免费，摊位级（简化）|

利息机制直接复制 The Bazaar，因为它是控制经济节奏的经典设计：
- 存 5 金币 → 每夜多 1 金币
- 上限 +3 → 存 15 和存 30 效果一样，防止无限储蓄
- 玩家必须在"花钱扩阵容"和"存钱吃利息"之间做选择

## 六、前 5 天商店流程（借鉴 The Bazaar 的教学曲线）

### 6a. The Bazaar 的前期节奏

The Bazaar 的前几天设计了一个很清晰的教学曲线：

```
Day 1-3: 学习基础物品 + 商人流程
Day 4-6: 知道什么是"构筑方向"，开始找协同
Day 7-9: 高阶物品 + 钻石改造，构筑定型
Day 10:  最终决战
```

### 6b. 我们怎么适配——商店第 1-5 夜流程

**第1夜——引导式商店**
- 只开 2 个摊位，固定展示火苗使和水滴使
- 初始金币 3💰，刚好买一个（教学"买英雄"）
- 引导弹窗："推荐购买火苗使，前排近战，适合第一关"
- 这一夜对应 The Bazaar 的"第一场比赛让你赢"——不让你犯错

**第2夜——合成教学**
- 新增第 3 个摊位
- 故意在摊位上放第二个火苗使 → 引导玩家合成升阶
- 引导弹窗："火苗使还有一位同伴……买下她可以合成升阶！"
- 对应 The Bazaar 的"告诉你物品可以升级"阶段

**第3夜——精英过渡**
- 出现第一个 2 阶摊位
- 十字使或召唤使作为"稀有品"随机出现
- 无强制引导，只提示"今晚有好货"
- 对应 The Bazaar 的"开始看见高阶物品"阶段

**第4夜——构筑扩展**
- 开第 4 个摊位
- 高概率出召唤使（配合第4天技能教学）
- 2阶英雄开始稳定出现
- 对应 The Bazaar 的"确定构筑方向"阶段

**第5夜——构筑成型**
- 4 个摊位全部开放
- 1阶/2阶混合，3阶开始有小概率出现
- 刷新和冻结在此夜变得重要
- 对应 The Bazaar 的"双商人双刷新"关键节点

## 七、代码层面的桥接——不需要动现有战斗系统

### 7a. 现有的东西可以直接用

| 现有代码 | 做什么 | 在大巴扎系统里的角色 |
|---|---|---|
| `UNIT_DEFS` | 英雄定义 | 每个英雄追加 tags 数组 |
| `addOwnedUnit()` | 购买英雄 | 购买按钮回调 |
| `buildHeroesFromUnits()` | 从单位重建 G.heroes/slots | closeShop 时自动调用 |
| `mergeUnits()` | 合成升阶 | 买同名英雄时自动触发 |
| `toggleUnitActive()` | 上阵/下阵 | 阵容管理操作 |
| `G.shopState` | 商店状态 | 新增 stalls 数组 |
| `finishMonsters()` | 波次结束 → 进商店 | 改调 openShopWithStalls() |

### 7b. 真正需要新增的

1. **标签系统**：在 `UNIT_DEFS` 里给每个英雄加 `tags` 数组
2. **摊位生成器**：`generateStalls(day)` → 返回摊位列表
3. **摊位刷新函数**：`refreshStall(idx)` → 按标签过滤 + 排除已拥有
4. **商店 UI 改造**：从 2 区布局 → 多摊位卡片布局

不需要动 `G.heroes`、`G.slots`、`useSlot`、`monsterAct`、`settleExplosions`、`previewGrid`——战斗系统完全不受商店改造的影响。

## 八、我们不借什么（有意差异）

| The Bazaar 机制 | 为什么不借 | 我们的替代 |
|---|---|---|
| PvP 对战匹配 | 我们的核心是回合战术棋盘 + 城堡推进 PvE | 城堡 HP 100 推进线 |
| auto-battle | 我们的乐趣是手动走位+元素叠层 | 保持回合制手动 |
| 物品/武器/技能混合 | 首版收敛，英雄够组合就不需要 | 只卖英雄，英雄=行动槽容器 |
| 钻石品级上限 4 级 | 3 阶已经能覆盖 Demo 阶段的战斗曲线 | 先做 1阶/2阶，3阶预留 |
| 商品级冻结 | 摊位少，摊位级冻结成本更低 | 摊位级冻结 |

## 九、总结：借鉴清单

```
借来的：                   落地位置：
── ─ ─ ─ ─ ─ ─ ─ ─ ─     ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
多摊位标签过滤             UNIT_DEFS.tags[] + generateStalls()
品级统一定价（同品级同价）   price = {1:2, 2:4, 3:6}
刷新递增                   refreshStall() fee = min(count, 3)
冻结                       stall.frozen = true
利息机制                   gold += floor(savedCoins/5)
角色即商品                 首版只卖英雄
日数品级曲线                calcShopTier(day)
第10天终局                 城堡 HP → 0

自己加的：
── ─ ─ ─ ─ ─ ─ ─ ─ ─
全局刷新（3金币）          作为"换方向"的出口
英雄可出售（返品级价-1）   给买错/换方向的容错
摊位级冻结（非商品级）      降低操作复杂度
摊位首次免费刷新           鼓励逛每个摊位
城堡 HP 每日扣减节奏       替代 PvP 胜负的进程感
