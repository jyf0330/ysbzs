## 一、新版第1天核心变化

和上一版对话纪要相比，这一版有 6 个**根本性变化**：

| # | 变化 | 旧设计 | 新设计 |
|---|---|---|---|
| 1 | **最多4个英雄** | 2个上阵位 | 场上最多4个 |
| 2 | **定价=格数×品级** | 品级统一价 | 青铜1格=2, 2格=4, 3格=6 |
| 3 | **范围英雄打2目标** | 范围=多格子 | 水水使每击给2个目标各叠1层 |
| 4 | **白银=每次叠2层** | 没细说 | 白银火苗使每击叠火2层 |
| 5 | **新英雄直布战场** | 放备战区 | 不满4人时直接上阵 |
| 6 | **元素叠层先于结算** | 已有 | 再次强调：所有英雄打完才统一结算 |

## 二、代码差距逐条对照

### 差距1：最大英雄数 2→4

**当前代码：**
```js
// syncUnitsToHeroes (line 1300)
const active=G.ownedUnits.filter(u=>u.active).slice(0,2);  // ← 写死2
const hid=ui===0?'ha':'hb';  // ← 只有 ha/hb
```

**影响范围：**
- `syncUnitsToHeroes()` — 英雄ID硬编码 `ha`/`hb`
- `buyUnit()` — `activeCount>=2` 判断写死
- `toggleUnitActive()` — `activeCount>=2`
- CSS — `.hma` `.hmb` 只有2个颜色，需要 `.hmc` `.hmd`
- 右侧面板英雄状态区 — 只渲染2个
- 全部 `G.heroes[slot.hid]` 引用 — 这部分是动态的，不硬编码 👍

**改法：**
```js
// 1. hid 改为动态生成
const hid = `h${ui}`;  // h0, h1, h2, h3

// 2. max 改为可配置
const MAX_ACTIVE_HEROES = 4;

// 3. CSS 加 .hm2 .hm3
.hm0{background:#1565C0}  // 蓝（原 ha）
.hm1{background:#2E7D32}  // 绿（原 hb）
.hm2{background:#9C27B0}  // 紫
.hm3{background:#E65100}  // 橙
```

### 差距2：价格 = 格数 × 品级倍率

**当前代码：**
```js
// SHOP_PRICE_CONFIG (line 541)
unitPrice: { tier1:3, tier2:6 }
// 所有1阶一个价，所有2阶一个价
```

**新设计：**
| 品级 | 1格 | 2格 | 3格 |
|---|---|---|---|
| 青铜 | 2 | 4 | 6 |
| 白银 | 4 | 8 | 12 |
| 黄金 | — | — | — |

```js
// 新定价表
unitPrice: {
  青铜: { 1:2, 2:4, 3:6 },
  白银: { 1:4, 2:8, 3:12 },
  黄金: { 1:6, 2:12, 3:18 },
}

// 价格 = unitPrice[grade][gridCount]
// gridCount = 该英雄主要攻击形状的格子数
```

**影响范围：**
- `SHOP_PRICE_CONFIG.unitPrice`
- `genShop()` 里赋值 `cost`
- `buyUnit()` 扣金币
- 每个 `UNIT_DEFS` 需要新字段：`grade: '青铜'`, `gridCount: 1`

### 差距3：品级影响叠层数

**新规则：**
- 青铜英雄每击叠 1 层
- 白银英雄每击叠 2 层

**当前代码已有基础：**
```js
// slot.tier 控制倍率
// TIER_MULT = {1:1, 2:2, 3:3, 4:4}
// 在 useSlot → addEl 中 tier 决定叠几层
```

**需要确认：** 品级（青铜/白银/黄金）和 tier（1/2/3）是不是同一个东西？如果是，当前 tier 已经实现了叠层倍率。

### 差距4：范围英雄打多目标

**新设计：** 水水使的"范围"不是多格子形状，而是**每击同时给2个不同目标各叠1层**。

**当前代码做不到：**
```js
// useSlot 当前逻辑：遍历 atkCells → 每个格子 addEl 一次
// 不支持"同一击命中多个目标各叠层"
```

**需要新增：**
- 槽位新属性 `targetCount: 2`（该槽每击可命中N个目标）
- `useSlot` 中：如果 slot.targetCount > 1，在攻击范围内选最近的N个怪物各自叠层

### 差距5：新英雄直布战场

**当前代码：**
```js
// buyUnit (line 1434)
const newUnit=addOwnedUnit(item.defId, 
  activeCount===0?{r:10,c:1}:activeCount===1?{r:11,c:1}:null);
```

**新设计：** 右侧，HP 100，玩家攻击目标，HP≤0→胜利

**需要改：**

- 城堡不能被英雄/怪物站在上面（已有 `cellFree` 检查）
