## 第4步：定价系统

### 当前代码

```js
// index.html line 541
const SHOP_PRICE_CONFIG = {
  unitPrice: { tier1:3, tier2:6 },   // ← 删掉
  ...
}

// genShop() 里
G.shopItems.units.push({id:..., defId, cost:SHOP_PRICE_CONFIG.unitPrice.tier1});
```

### 改成

```js
// 品级基础价（每个定价档位的单价）
const GRADE_BASE = { 青铜:2, 白银:4, 黄金:6, 钻石:8 };

// 定价函数
function calcUnitPrice(def) {
  return GRADE_BASE[def.grade] * def.priceTier;
}

// genShop() 里
const def = UNIT_DEFS[defId];
G.shopItems.units.push({id:..., defId, cost:calcUnitPrice(def)});
```

### 需要删除的代码

| 位置 | 删什么 |
|---|---|
| `SHOP_PRICE_CONFIG.unitPrice` | 整行删掉 |
| `UNIT_DEFS.*.cost` | 每个英雄的 `cost:2` 字段可以删，因为价格由 `calcUnitPrice` 动态算 |

### 价格表

| 品级 \ 档位 | 1档 | 2档 | 3档 |
|---|---|---|---|
| 青铜 | 2 | 4 | 6 |
| 白银 | 4 | 8 | 12 |
| 黄金 | 6 | 12 | 18 |
| 钻石 | 8 | 16 | 24 |

## 第6步：水水使的攻击方式（不需要改 useSlot）

水水使的特殊性在于它的**攻击形状**，不在叠层逻辑。

```
水水使 SD[4] = 3格拐角（cells:[[0,1],[1,1]]），方向 right →
  格子1：(hero.r+0, hero.c+1)  ← 命中怪物3 → 叠水层
  格子2：(hero.r+1, hero.c+1)  ← 命中怪物4 → 叠水层
```

和火苗使的 `useSlot` 逻辑完全一样：
1. `atkCells()` 算出攻击覆盖的格子
2. 每个格子分别 `addEl()` 叠层
3. 结算时各自独立计算伤害

**不需要任何新代码。** 水水使就是用了 2 格形状的英雄。
