## 二、差距逐条分析

### 差距A：召唤系统（元素小灵）

**设计：**
- 召唤使每回合主动召唤1只元素小灵
- 元素小灵：HP 6/6, ATK 1, 自动向最近怪物移动+攻击
- 不吃玩家操作槽，全自动
- 死亡后不留到下一天

**当前代码：** ❌ 完全不存在

**需要新增：**
```
1. G.summons = []  存储场上召唤物
2. summonPet(hero, pos)  在指定位置创建召唤物
3. petAct(pet)  召唤物AI：找最近怪物→移动→攻击
4. endPlayerTurn 后自动执行所有召唤物行动
5. render() 渲染召唤物到棋盘
```

**召唤物数据结构：**
```js
G.summons = [{
  id: 'pet_1',
  hp: 6, maxHp: 6,
  atk: 1,
  pos: {r:8, c:5},
  ownerId: 'u_召唤使',
}]
```

**召唤物AI（简化版）：**
```js
function petAct(pet) {
  const nearest = findNearestMonster(pet.pos);
  if (!nearest) return;
  const dist = abs(nearest.pos.r - pet.pos.r) + abs(nearest.pos.c - pet.pos.c);
  if (dist === 1) {
    // 相邻 → 攻击
    nearest.hp -= pet.atk;
  } else {
    // 有距离 → 移动1格
    moveToward(pet, nearest.pos);
  }
}
```

### 差距B：被动光环系统

**兽语使设计：** 场上所有召唤物 HP+2, ATK+1

**灵爆使设计：** 召唤物死亡时，死亡格留下火2层

**当前代码：** ❌ 完全不存在

**需要新增：**
```js
// 1. 英雄增加 passive 字段
UNIT_DEFS.beast_whisper: {
  ...
  passive: { type:'summon_buff', hpBonus:2, atkBonus:1 }
}

// 2. 召唤物创建时应用光环
function applyAuras(pet) {
  G.ownedUnits.filter(u=>u.active).forEach(u => {
    const def = UNIT_DEFS[u.defId];
    if (def.passive?.type === 'summon_buff') {
      pet.maxHp += def.passive.hpBonus;
      pet.hp += def.passive.hpBonus;
      pet.atk += def.passive.atkBonus;
    }
  });
}

// 3. 召唤物死亡时检查灵爆使
function onPetDeath(pet) {
  G.ownedUnits.filter(u=>u.active).forEach(u => {
    const def = UNIT_DEFS[u.defId];
    if (def.passive?.type === 'death_element') {
      addEl(pet.pos, def.passive.el, def.passive.layers);
    }
  });
}
```

**光环类型（草案）：**

| 类型 | 效果 | 英雄 |
|---|---|---|
| summon_buff | 召唤物HP+2, ATK+1 | 兽语使 |
| death_element | 召唤物死亡留火2层 | 灵爆使 |
| 后续扩展 | summon_count, element_aura... | — |
