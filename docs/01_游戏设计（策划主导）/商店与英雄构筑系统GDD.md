## 0. 本 GDD 的核心决策（必读）

> **遗物（relic）作为独立系统被彻底砍掉。**
>
> 理由：**英雄的被动 / 主动效果本身就是「遗物」**——它们提供全局/构筑级的特殊效果。再单设一套遗物物品会与英雄能力重叠、增加隐蔽规则与平衡负担。
>
> **商店只卖英雄。** build 深度来自：英雄能力联动 + 同名合成升阶 + 上阵组合，而非独立装备/遗物条。

这条与 deep-interview 一致（A3：可流通物品=英雄本身；偏移条目：商店只卖英雄）。

---

## 1. 系统目的

在 roguelite 壳（S3）里，商店是**节点之一**（与战斗/精英/事件/Boss 并列）。本系统定义：

- 商店在 roguelite 结构中的角色与节奏。
- 「只卖英雄」下的 build 深度从哪来。
- 英雄能力如何承接原本设想给遗物的「放大引擎/连锁」职能。
- 与现有代码（`SHOP_POOLS`/`UNIT_DEFS`/`passive`/合成）的衔接与**遗物代码清理**。

---

## 2. 设计支柱对应

| 全局支柱 | 本系统承接 |
|---|---|
| 物品=英雄（SAP） | 商店唯一商品是英雄；能力即构筑 |
| 随机只在局外 | 商店刷新/英雄池是局外随机，战斗内不变 |
| 连锁是高潮 | 「放大连锁」的效果改由**英雄被动**提供，不再由遗物提供 |

---

## 3. 商店只卖英雄

### 3.1 商品 = 英雄卡

- 商品列表只含 `UNIT_DEFS` 英雄（按 `SHOP_POOLS[day_phase]` + tier 过滤）。
- 购买 → 上阵（≤上限）或备战；同名自动/手动合成升阶（已实现 `mergeUnits`）。
- **不再有**：遗物商品、独立攻击方块商品、装备。

### 3.2 强化品（保留，定位收窄）

- 现有「元素+1 / 金币袋」等强化品**可保留**为轻量经济/微调，但：
  - 不承担 build 主力（那是英雄能力的职责）。
  - 不得演化成「变相遗物」（避免回到被砍掉的老路）。
- 若强化品与英雄能力职能重叠，优先削强化品。

### 3.3 刷新 / 冻结 / 经济

- 沿用现有 `genShop` / `getShopPoolKey` / 冻结集合。
- 经济细节（固定收入、利息、击杀奖励上限）见 [`商店系统策划方案.md`](商店系统策划方案.md) §5，**仅经济部分仍有效**；遗物部分作废。

---

## 4. 英雄能力 = 原「遗物」职能

原 roguelite 草案设想用遗物提供「放大连锁/引擎」的全局效果（例：连锁每段 +X 伤害、引爆不清首层）。这些职能**全部改挂到英雄被动/主动**：

| 原遗物设想 | 改由英雄承接（示例） |
|---|---|
| 引爆伤害 +X（`old_fuse`） | 火系英雄被动：空格引爆 +X（设计为某英雄的 passive） |
| 连锁额外波及斜向 | 风/水系英雄被动：引爆波及形状扩展 |
| 召唤越多越强 | 召芽灵阶位（见成长 GDD §3） |
| 治疗放大 | 泉泉灵/绒语灵被动（已实现 buffAllSummons） |

### 4.1 能力分层

| 层 | 载体 | 现状 |
|---|---|---|
| 局部 | 行动槽（el/sn/dir/tier） | ✅ |
| 单位级 | `UNIT_DEFS.passive`（绒/爆/分） | ✅ S1 |
| 构筑级（原遗物职能） | 英雄被动/主动技能 | 📋 S3 扩展 |

### 4.2 约束

- 构筑级能力仍遵守反支柱：**不引入战斗内随机**；效果必须可预览、可结算。
- 能力数量克制：优先把「放大连锁/引擎」做深，而非铺很多零散被动（沿用 §10 旧方案「容易做复杂」的警告）。

## 6. 与现有代码的衔接 + 遗物清理

### 6.1 可直接复用

- `SHOP_POOLS` / `getShopPoolKey` / `genShop` / `UNIT_DEFS` / `mergeUnits` / `syncUnitsToHeroes`。

### 6.2 ⚠ 代码-文档差异（需后续代码任务清理）

代码当前**仍存在遗物系统**，与本决策冲突：

| 位置 | 内容 | 处置 |
|---|---|---|
| `index.html` `G.relics:[]` | 遗物数组状态 | 移除或冻结不再写入 |
| 商店商品 `{type:'relic',relicId:'old_fuse',cost:5}` | 卖遗物 | **删除该商品** |
| `settleExplosions` `relicBonus`（old_fuse +1） | 空格爆炸加成 | 改为英雄被动；删除遗物分支 |
| `buildPreviewGrid` `relicSpaceBonus` | 预览同步 | 同步删除 |
| `buyItem` `item.type==='relic'` 分支 | 购买遗物 | 删除 |

> 处置原则：属**行为变化 + 代码改动**，需走 TDD（先改/删测试到 RED，再清理）。本 GDD 只记录，不在文档轮次里改 `index.html`。

## 8. 明确不做

- 遗物 / 装备 / 独立攻击方块商品（**永久砍掉**）。
- 战斗内随机化商店效果。
- 复杂经济流派（贷款/投资）。
