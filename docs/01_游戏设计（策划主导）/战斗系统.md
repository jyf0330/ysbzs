# 战斗系统.md

## 1. 战斗核心定位

本游戏是严肃战斗，不是纯表现型战斗。

玩家每回合需要通过英雄位置、行动点、攻击方块、元素叠层、元素引爆和怪物属性克制，提前预判本回合结算结果。

核心目标：
玩家在行动前能看懂：
- 攻击会落在哪些格子
- 会生成什么元素
- 会叠到第几层
- 是否触发引爆
- 会炸到哪些目标
- 怪物会受到多少伤害
- 哪些怪物会死亡
- 剩余怪物是否会反击英雄

## 2. 元素类型

游戏使用四种元素：

- 风
- 火
- 水
- 土

元素采用循环克制关系。

暂定循环顺序：

风 → 火 → 水 → 土 → 风

后一个元素可以克制/引爆前一个元素。

所以克制关系为：

- 火克风
- 水克火
- 土克水
- 风克土

例如：
火层上落下水元素时，水克火，触发火层引爆。

## 3. 元素生成规则

每个行动点发动攻击时，会在攻击范围内生成对应元素。

如果攻击命中怪物：
- 直接对该怪物造成命中伤害
- 直接命中只作用于被打中的那个怪物

如果攻击没有命中怪物：
- 仍然会在攻击落点生成元素层
- 元素层可以后续被继续叠加或引爆

## 4. 元素叠层规则

同一个格子可以叠元素层。

如果新落下的元素和当前格子顶层元素相同：
- 层数 +1
- 继续累计伤害

例如：

火层第1层：1伤害  
火层第2层：2伤害  
火层第3层：3伤害  
火层第4层：4伤害  
火层第5层：5伤害  
火层第6层：6伤害  

累计基础伤害：

1层 = 1  
2层 = 1 + 2 = 3  
3层 = 1 + 2 + 3 = 6  
4层 = 1 + 2 + 3 + 4 = 10  
5层 = 1 + 2 + 3 + 4 + 5 = 15  
6层 = 1 + 2 + 3 + 4 + 5 + 6 = 21  

第一版建议最高先做到 6 层。

## 5. 引爆触发规则（行动/结算两阶段）

### 5a. 行动阶段（useSlot）

玩家使用行动点时，进入**行动阶段**：

- 元素叠层写入 `elementCells` 各格子槽（独立追踪每种元素的层数 / 待结算伤害 / 是否达到引爆阈值）
- 伤害只计算为"预览伤害"，**不立即扣怪物 HP**
- 棋盘显示当前元素状态，方便玩家预判

引爆阈值（默认）：

> **G.explosionThreshold = 3**

当同一元素在同一格叠到 ≥ 3 层时，该槽 `willExplode = true`，进入"待结算引爆"状态。

### 5b. 结算阶段（endPlayerTurn → settleExplosions）

玩家按下"结束回合"后：

1. 调用 `settleExplosions()`
2. 遍历所有格子的所有元素槽
3. 凡 `willExplode = true` 的槽，将 `previewDamage` 真正用 `dealDmg()` 扣怪物 HP
4. 清空对应槽的层数和预览伤害

结算后怪物才真正掉血。

### 5c. 旧兼容路径（addEl 跨元素反应）

`addEl()` 函数仍保留跨元素即时引爆逻辑（水落火格 → 立即 doExplode）。这是专门用于：

- 怪物踩踏元素格时触发引爆（monsterAct）
- 测试中直接调用 addEl 的场景
- 特殊即时爆炸道具（待实现）

⚠️ 普通行动点（useSlot）**不走此路径**，真实伤害必须经由 settleExplosions() 统一结算。



例如：

当前格子已有：

火层×5  
累计基础伤害 15  

此时水元素落到该格子：

水克火  
触发火层引爆  

引爆伤害按"被引爆的火层"计算，不按新来的水层计算。

也就是：

火层×5 的累计伤害 = 15  
触发后造成 15 点引爆基础伤害  

## 6. 引爆后重置规则

引爆后，被引爆的旧元素层清空。

触发引爆的新元素会成为新的第1层。

例如：

火层×5  
↓  
水落下，触发火层引爆  
↓  
火层清空  
↓  
当前格子变成水层×1  

下一次如果再落水，就是水层×2。  
下一次如果落土，因为土克水，则触发水层引爆。

## 7. 直接命中与引爆范围

直接攻击命中怪物时：

- 只打中当前目标怪物
- 不自动波及周围怪物

引爆发生时：

- 以引爆格为中心
- 按十字范围炸开
- 影响中心格、上、下、左、右

十字范围内的怪物都会受到引爆伤害。

所以区别是：

直接命中 = 单体  
元素引爆 = 十字范围  

## 8. 怪物属性规则

第一关怪物没有属性。

第一关重点教学：
- 行动点攻击
- 元素生成
- 元素叠层
- 元素引爆
- 十字范围爆炸
- 伤害预览

第二关开始出现属性怪物。

怪物可以拥有元素属性：

- 风属性怪
- 火属性怪
- 水属性怪
- 土属性怪

当攻击元素或引爆元素克制怪物属性时，伤害翻倍。

克制倍率：

克制：×2  
不克制：×1  

## 9. 伤害计算规则

单层基础伤害 = 当前层数

例如第3层造成 3 点基础伤害。

累计层伤害 = 1 + 2 + 3 + ... + 当前层数

例如火层×5：

1 + 2 + 3 + 4 + 5 = 15

如果触发引爆，则按被引爆元素层的累计伤害计算。

如果目标怪物属性被克制，则最终伤害 ×2。

## 10. 战斗预览要求

所有元素叠层、直接伤害、引爆伤害、怪物剩余血量、怪物死亡状态，都必须在行动前预览。

玩家需要一眼看到：

- 当前格子是什么元素
- 当前是第几层
- 累计伤害多少
- 新元素落下后是叠层还是引爆
- 引爆后十字范围会炸到谁
- 哪些怪物会死亡
- 哪些怪物还会反击

## 11. 怪物行为预览是核心玩法数据

### 决策依据

怪物不是被动靶子。怪物每回合有固定行动点（当前 3 点），会自主移动并在攻击范围内攻击最近英雄。

玩家每回合决策（摆放英雄、调整行动块方向、判断是否引爆、决定是否结束回合）时，必须提前知道：

- 怪物会不会走过来
- 怪物会停在哪里
- 怪物能不能攻击英雄
- 怪物会攻击哪个英雄（→A / →B）
- 英雄预计会掉多少血（⚠-N）
- 哪些格子属于怪物威胁范围（红色高亮 .mw-atk）

### 显示规则

怪物行为预览**无条件、始终显示**，不依赖：
- 玩家是否选中英雄
- 玩家是否选中行动槽
- 玩家是否点击或悬停怪物
- 玩家是否攻击命中怪物

### 当前实现（HUD 展示要求）

| 信息 | 显示位置 | 触发条件 |
|---|---|---|
| 怪物移动路径 M1/M2/M3 | 路径格右上角徽章 | 无条件 |
| 怪物停留格 | .mw-final（黄色描边）| 无条件 |
| 怪物攻击范围 | .mw-atk（红色描边）| 无条件 |
| 怪物攻击目标 | 怪物卡 .mm-atkinfo（→A 蓝 / →B 绿）| 无条件 |
| 怪物造成伤害 | 怪物卡 .mm-atkinfo（-N 红）| 无条件 |
| 无法攻击标记 | 怪物卡 .mm-atkinfo（·灰色）| 无条件 |
| 英雄预计受伤 | 英雄卡 .hm-inc（⚠-N 紫色）| 无条件 |

### 优先级

界面空间不够时，优先级由高到低：

1. 英雄当前血量
2. 怪物当前血量
3. 怪物预计攻击目标（→A / →B）
4. 英雄预计受到伤害（⚠-N）
5. 怪物预计受到伤害
6. 怪物移动路径（M1/M2/M3）
7. 玩家行动块攻击范围
8. 元素层变化
9. 引爆提示
10. 装饰性 UI
