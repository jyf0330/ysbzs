## 推荐目录

```text
data/
  inbox/        # 新收到、待整理的数据
  raw/          # 原始只读数据，不手改
  staging/      # 清洗中、转换中、待确认的数据
  processed/    # 已整理、可直接被项目使用的数据
  reference/    # 规则表、字典、映射、配置样例
  exports/      # 导出结果、报表、快照
  archive/      # 已过期但需要留存的数据
```

## 分类规则

- 新进入项目、暂时还没判断归属的数据：先放 `data/inbox/`。
- 外部原始文件、抓取结果、导入源：放 `data/raw/`。
- 正在清洗、重命名、拆分、合并的数据：放 `data/staging/`。
- 已被项目代码、流程、工具稳定消费的数据：放 `data/processed/`。
- 枚举、映射、说明型表格、模板样例：放 `data/reference/`。
- 脚本或流程产出的结果文件：放 `data/exports/`。
- 已完成历史批次、旧版本、冻结快照：放 `data/archive/`。

## 根目录清理规则

- 本规范只定义归档策略，不在未确认前移动现有文件。
- 迁移任何根目录数据文件前，先确认是否被 `index.html`、`test.js` 或文档引用。
- 不能立即判断归属时，先进入 `data/inbox/`。
- 迁移后在本规范中补记录：来源、用途、当前分类、后续处理阶段。
