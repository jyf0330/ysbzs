# 美术风格指南 · 元素背包史

## 文档状态

- 状态：草稿（Demo v0.1 羊皮纸手绘风）
- 最后更新：2026-06-01

## 整体风格

- 风格定位：浅色羊皮纸战术桌 / 手绘线稿风，无图片资源，纯 CSS + Emoji 呈现。
- 色调：暖色纸面底色（`#f2eadc`），深棕色文字（`#4a3f35`），低饱和柔和配色。
- 氛围：童话战棋 / doodle tactics / 手账风；主界面要像摊开的战术地图，而不是普通网页后台。
- 字体：优先日系圆体/手写体（Hiragino Maru Gothic ProN、Klee、YuKyokasho），回退到系统无衬线。

## 主界面成品化要求（2026-06-01）

- 棋盘必须是第一视觉焦点；左右栏和底部日志只能辅助，不得与棋盘等权抢视线。
- Debug 可以常驻左侧，但必须包装为“调试书记板”：低对比、只读、开发感可辨识但不破坏游戏画面。
- 行动槽必须像“符文行动卡”：元素色带、编号徽章、形状图、方向与执行按钮分层明确，不做普通表单列表。
- 顶栏使用卷轴状态栏语义；底栏使用战报条语义；“开始战斗”是唯一最高优先级主按钮。
- 棋子牌面要比调试短字更有存在感：英雄、怪物、召唤物、城堡使用实体牌面边框和元素/阵营色，避免针尖小图标。
- 可使用 CSS 渐变、双线边框、纸面内阴影、压印按钮、角饰与坐标轴；不要引入外部图片依赖。
- 可读性优先于装饰：顶栏状态、行动卡主信息、方向按钮、执行按钮、底部主按钮必须先清楚可读、可点，再谈材质；小字只允许用于 debug、辅助说明和次级日志。
- AI 战斗按钮允许使用轻量“推演中”脉冲，但只表达系统正在行动；不要做持续闪烁、强光或会抢走棋盘注意力的装饰动画。禁用态必须仍能读出文案。

## 线条风格

- 边框统一使用细实线或虚线（1-2px），颜色为暖棕色系。
- 无 box-shadow 发光效果；选中状态用虚线描边替代光晕。
- 格子间隙用棋盘底色透出，模拟纸上画线感。
