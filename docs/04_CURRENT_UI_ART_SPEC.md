# 04_CURRENT_UI_ART_SPEC · UI 布局、美术风格与交互规范

## 整体风格

- **风格定位**：浅色羊皮纸战术桌 / 手绘线稿风。
- **实现方式**：纯 CSS + Emoji 呈现，无外部图片资源依赖。
- **色调**：暖色纸面底色（`#f2eadc`），深棕色文字（`#4a3f35`），低饱和柔和配色。
- **氛围**：童话战棋 / doodle tactics / 手账风。
- **字体**：优先日系圆体/手写体（Hiragino Maru Gothic ProN、Klee、YuKyokasho），回退系统无衬线。

## 主界面布局（2026-06-01）

固定 1280×720 战术桌布局，运行时缩放适配浏览器窗口。

```
┌────────────────────────────────────────────────────────────────────┐
│ 顶部卷轴状态栏：标题 / 阶段 / 天数 / 波次 / 金币 / 双城堡 / 商店     │
├──────────────┬──────────────────────────────────────┬──────────────┤
│ 左侧调试书记板 │ 中央棋盘舞台（#board-stage + #board） │ 右侧符文行动卡 │
│ #debug-dock  │ 8×8 棋盘 + 坐标轴 + 棋子牌面          │ #asl/.as-card │
├──────────────┴──────────────────────────────────────┴──────────────┤
│ 底部战报条：战斗日志 + 玩家回合状态 + 主操作按钮                    │
└────────────────────────────────────────────────────────────────────┘
```

- 棋盘是首屏第一视觉焦点。
- 左右栏不得比棋盘更抢眼。
- Debug 面板开发期可读但降权为只读侧栏。

## 可读性基线（1280×720）

| 区域 | 要求 |
|---|---|
| 顶栏状态 | 阶段/天数≥13px；金币≥14px |
| 行动卡文字 | 主文字≥15px；层数≥12px |
| 方向按钮 | ≥22×22px |
| 底部主按钮 | 高度≥44px |
| 棋盘提示 | ≥11px，完整露出 |

## 当前 UI 重点

- 棋盘必须是主视觉
- 始终显示玩家攻击预览和怪物行为预览
- 行动槽使用"符文行动卡"形态
- AI 战斗按钮语义为"AI 战斗"，推演中有轻量脉冲
- 战斗日志从新到旧排列
- 禁用高饱和、复杂页游感 UI

## 交互规范

- 按钮默认态：高对比底色 + 明确文字
- 禁用态：降低视觉突出，禁止触发核心动作
- 选中态：持续高亮直到切换或阶段结束
- 操作成功：更新对应 HUD、棋盘预览、日志
- 操作失败：说明不可操作原因
- 模式切换：进入商店/怪物行动时同步隐藏不适用操作

### 禁止绑定显示条件

怪物行为预览不得依赖：选中英雄、选中行动槽、鼠标悬停、点击怪物、玩家攻击命中。
