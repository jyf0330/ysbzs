## 【旧商店系统输入输出对照】

### 1. 初始状态

| 字段 | 初始值 | 来源 |
|---|---|---|
| `G.gold` | 5 | `initGame()` |
| `G.phase` | `'PLAYER'` | `initGame()` |
| `G.shopItems` | `[]` | `initGame()` |
| `G.shopEls` | `[]` | `initGame()` |
| `G.backpack` | `[]` | `initGame()` |
| `G.heroes` | `{ ha: {...}, hb: {...} }` — 写死两个英雄 | `initGame()` |
| `G.slots` | 6 个槽，ha×3 + hb×3，默认全 fire·十字5格·tier1 | `initGame()` |

### 2. 进入商店

- **触发**: `finishMonsters()` 中 `round > maxRound` 或所有怪物死亡
- **输入**: `G.gold`(当前), `G.wave`(当前)
- **动作**: `G.gold += 3` → `G.phase = 'SHOP'` → `openShop()`
- **openShop()**: `genShop()` + `renderShop()` + 显示 `#so`
- **genShop()**: 
  - 生成 3 个 `shopItems`（攻击方块）：随机 el(4选1) + 随机 sn(1-20) + tier=1 + cost(1-4,基于sn.n)
  - 生成 0-3 个 `shopEls`（裸元素）：30%概率0个，20%1个，20%2个，30%3个，每个随机 el + cost=1
- **输出**: `G.shopItems.length === 3`，`G.shopEls.length ∈ [0,3]`

### 3. 购买攻击方块

- **触发**: 点击商品 → `buyToBackpack(itemId)`
- **输入**: `G.gold` ≥ cost, `G.backpack.length` < 64
- **动作**: `G.gold -= cost` → `G.backpack.push(item)` → `G.shopItems` 移除该项
- **金币不足**: 弹窗提示，不修改任何状态
- **背包满**: 弹窗提示"背包已满"，不购买
- **输出**: 金币正确扣除，商品移入背包，商品从商店列表消失

### 4. 装备到行动槽

- **触发**: 背包物品点击"→槽N" → `bpEquip(bpId, slotIdx)`
- **输入**: 背包中存在对应 bpId
- **动作**: 从背包删除 → 覆盖 `G.slots[slotIdx]` 的 el/sn/tier → `used = false`
- **输出**: 槽属性改变，背包数量-1

### 5. 背包合成

- **触发**: 两个同 el+sn+tier 物品在背包中 → `bpCombine(id1, id2)`
- **限制**: tier < 4
- **动作**: 删除两个物品 → 创建新物品 tier+1
- **输出**: 背包数量-1，新物品 tier = 旧+1

### 6. 购买裸元素

- **触发**: 点击元素商品 → `buyEl(elId)`
- **输入**: `G.gold` ≥ 1
- **动作**: 扣 1 金币 → 从 shopEls 移除 → `placeEl(el)` 在 7-12 行找空位放元素
- **输出**: 棋盘上出现元素，金币-1

### 7. 刷新商店

- **触发**: `refreshShop()`
- **输入**: `G.gold` ≥ 1
- **动作**: `G.gold -= 1` → `genShop()` → render
- **金币不足**: 弹窗提示，不刷新
- **输出**: 3 个新 shopItems + 新 shopEls（随机）

### 8. 关闭商店

- **触发**: `closeShop()` 按钮
- **输入**: 无校验
- **动作**: 隐藏 `#so` → `G.wave++` → `G.round=1` → `G.hitCount=0` → `G.phase='PLAYER'` → 重置所有 slot.used=false → `spawnWave(G.wave)`
- **输出**: 新波次开始

### 9. 行动槽间合成

- **触发**: `combineSlots(i, j)` — 商店界面两个同 el+sn+tier 槽合成
- **动作**: 槽 i 升阶，槽 j 重置为 fire·1格·1号·1阶

### 10. 无冻结机制

- 当前系统没有 freeze/lock 功能

## 【旧商店系统测试草案（characterization tests）】

每条按格式：编号 / 名称 / 输入 / 旧输出 / 新目标 / 分类

### case_shop_001: 初始金币
- **输入**: `initGame()`
- **旧系统输出**: `G.gold = 5`
- **新系统目标**: `G.gold = 10`（参考 SAP DEFAULT_COIN_COUNT）
- **分类**: **B — 允许改变**。理由：旧商店 5 金币太少，新商店需要 10 金币匹配 SAP 模型
- **涉及函数**: `initGame()`
- **涉及字段**: `G.gold`

### case_shop_002: 初始英雄结构
- **输入**: `initGame()`
- **旧系统输出**: `G.heroes = { ha: {...}, hb: {...} }` — 写死两个
- **新系统目标**: `G.heroes` 由 `buildHeroesFromUnits()` 从 `G.activeUnits` 动态生成
- **分类**: **B — 允许改变**。理由：单位制是本次改造核心目标
- **涉及函数**: `initGame()`, `buildHeroesFromUnits()`(新)
- **涉及字段**: `G.heroes`, `G.activeUnits`

### case_shop_003: 初始 6 行动槽
- **输入**: `initGame()`
- **旧系统输出**: 6 个槽，ha×3 + hb×3，全 fire·十字5格(sn=12)·tier1
- **新系统目标**: 6 个槽，由 2 个上阵单位的 `actionSlots` 拼接，元素/形状随单位定义
- **分类**: **B — 允许改变**。理由：槽内容由单位决定而不是写死
- **涉及函数**: `initGame()`, `buildHeroesFromUnits()`
- **涉及字段**: `G.slots`

### case_shop_004: genShop 生成 3 个攻击方块
- **输入**: `genShop()`
- **旧系统输出**: `G.shopItems.length = 3`，全为随机攻击方块
- **新系统目标**: 商店商品分单位区和强化品区，不再生成裸攻击方块
- **分类**: **B — 允许改变**。理由：商店品类从攻击方块改为单位+强化品
- **涉及函数**: `genShop()`
- **涉及字段**: `G.shopItems`

### case_shop_005: 商品属性校验
- **输入**: `genShop()`
- **旧系统输出**: 每个商品 `el∈[fire/water/wind/earth]`, `sn∈[1,20]`, `tier=1`, `cost≥1`
- **新系统目标**: 单位商品含 `unitDefId/cost/level`，强化品商品含 `type/effect/cost`
- **分类**: **B — 允许改变**。商品数据结构完全变
- **涉及函数**: `genShop()`

### case_shop_006: buyToBackpack 扣金币并入背包

- **旧系统输出**: 金币扣 cost，商品进 `G.backpack`，从 `G.shopItems` 移除
- **新系统目标**: 买单位 → 进 `G.ownedUnits` 并放入上阵位/备战区。不再是"入背包"
- **分类**: **B — 允许改变**。购买目标从背包改为 ownedUnits
- **风险**: `G.backpack` 逻辑可能需要保留用于后续强化品

### case_shop_007: 金币不足不能购买
