# 《元素背包史》GDD v0.1

## 1. 游戏定位

```txt
类型：
格子战斗 + 背包整理 + 元素叠层 + 商店构筑

核心体验：
玩家在 13×13 棋盘上操控两个元素使，通过攻击方块、方向调整、元素叠层和商店购买，抵挡一波波怪物。

一句话玩法：
在格子战场里摆放和装备带元素的攻击方块，用 3 个行动点攻击、铺元素、引爆元素块，同时通过商店购买更好的方块构筑成长。
```

---

# 2. 核心卖点

```txt
1. 两个元素使开局，玩家可以直接拖动位置
2. 每回合有 3 个行动点
3. 每个行动点可装备不同攻击方块
4. 攻击方向块可调上、下、左、右
5. 攻击命中敌人时按第几次命中结算伤害
6. 攻击没命中敌人，也会在范围内生成元素
7. 元素格可以叠层，层数越高，引爆伤害越高
8. 元素格是独立场上物，可以阻挡怪物
9. 怪物被元素块挡住时，会攻击元素块并浪费本回合行动
10. 商店出售带元素的攻击方块和随机元素
11. 相同攻击方块可合成升级，伤害指数成长
```

```txt
开局英雄数量：2 个

```

```txt

```

规则：

```txt
1. 玩家拖动英雄到目标格
2. 如果目标格为空，可以放置
3. 第一版不计算移动力
4. 第一版不做路径阻挡
5. 后续可再加入移动限制
6. 英雄移动后保持选中状态，不取消选中
7. 英雄在新位置的攻击预览和怪物行动预览立即重算
8. 再次点击英雄本身可取消选中
```

# 6. 攻击方向块

## 6.1 方向

每个攻击方块可以调 4 个方向：

```txt
上
下
左
右
```

方向以当前英雄位置为基准。

## 6.2 调整方式

玩家可以每回合调整 3 个行动点的方向。

示例：

```txt
行动点1：向上
行动点2：向右
行动点3：向下
```

这样玩家可以实现：

```txt
上方铺元素
中间打怪
下方引爆
```

# 7. 攻击方块

## 7.1 攻击方块定义

攻击方块是一个带形状、元素、阶级的装备。

```txt
攻击方块 = 形状 + 元素 + 阶级
```

示例：

```txt
火·3格·7号·1阶
水·5格·25号·2阶
风·4格·12号·1阶
土·6格·31号·3阶
```

## 7.2 商品命名规则

商品名采用玩家能看懂的格式：

```txt
元素 + 几格 + 几号 + 几阶
```

示例：

```txt
火·5格·25号·1阶
水·3格·7号·2阶
风·4格·12号·1阶
土·6格·31号·4阶
```

含义：

```txt
火 = 元素属性
5格 = 占用 / 攻击形状由 5 个格子组成
25号 = 形状编号
1阶 = 当前等级
```

# 9. 玩家伤害规则

## 9.1 基础命中伤害

每回合玩家有 3 次行动点攻击。

如果攻击命中敌人，按命中顺序结算：

```txt
第1次命中：1点基础伤害
第2次命中：2点基础伤害
第3次命中：3点基础伤害
```

所以初始 3 次都命中时：

```txt
1 + 2 + 3 = 6
```

第一只教学怪血量为 6，正好被击杀。

## 9.2 阶级倍率

攻击方块分为 4 阶：

```txt
1阶：×1
2阶：×2
3阶：×4
4阶：×8
```

说明：

```txt
每升一阶，伤害翻倍。
```

## 9.3 元素克制倍率

四元素：

```txt
风
火
水
土
```

克制关系第一版建议：

```txt
水克火
火克风
风克土
土克水
```

克制时：

```txt
伤害 ×2
```

## 9.4 最终伤害公式

```txt
最终伤害 = 命中顺序伤害 × 阶级倍率 × 克制倍率
```

示例：

```txt
第3次命中
4阶攻击方块
属性克制

伤害 = 3 × 8 × 2 = 48
```

## 9.5 双向伤害预览系统

玩家回合内，棋盘会实时显示两类预览，无需任何额外操作。

### A. 英雄攻击预览（点击英雄触发）

```txt
点击英雄后，显示该英雄所有未使用攻击块的完整预判信息。
```

预览内容：

```txt
1. 所有未用攻击块的攻击格，以元素色半透明叠加显示
2. 怪物格右上角显示：最终元素 + 预计总伤害
   示例：火 -6
   含义：若释放所有攻击块，此怪预计损失 6 点生命值
3. 当多个攻击块命中同一怪物时，累计显示总伤害
4. 伤害按命中顺序（hitCount）依次递增累加，不修改真实状态
```

不同英雄独立计算：

```txt
英雄A的攻击块 → 仅计算英雄A的预览
英雄B的攻击块 → 仅计算英雄B的预览
点击哪个英雄，就显示那个英雄的预览
```

### B. 怪物行动预览（玩家回合全程显示）

```txt
不需要选中英雄，玩家回合期间棋盘始终显示怪物的下一步预判。
```

预览内容：

```txt
1. 橙色高亮：怪物下一步将移动到的格子（mw-mov）
2. 红色高亮：怪物下一步将攻击的目标格（mw-atk）
3. 英雄格左下角显示紫色徽章：该英雄预计受到的伤害
   示例：-1
   含义：若怪物攻击，此英雄将损失 1 点生命值
```

实时更新规则：

```txt
每次英雄移动后，怪物预览立即重算
怪物目标基于最近原则，随英雄位置变化而切换
若英雄移动未影响最短路径，怪物目标不变
```

### 预览仅为 UI 层计算

```txt
预览不修改真实血量
预览不修改真实 hitCount
预览不修改真实元素层数
预览不修改行动点使用状态
```

# 10. 元素叠层系统

## 10.1 生成元素

攻击方块释放后，会根据自己的元素，在攻击范围内生成元素。

```txt
火攻击方块 → 生成火元素
水攻击方块 → 生成水元素
风攻击方块 → 生成风元素
土攻击方块 → 生成土元素
```

## 10.2 命中和未命中

```txt
如果打到怪物：
造成伤害，并在对应格子生成元素。

如果没有打到怪物：
不会浪费，会在攻击范围内生成元素。
```

表现：

```txt
元素使施法后，攻击范围内落下对应颜色元素。
```

## 10.3 元素显示

格子上只显示：

```txt
元素 + 层数
```

示例：

```txt
火1
火2
火3
火4

水1
水2
土3
风5
```

不额外显示“谁能引爆它”。

## 10.4 多元素叠加显示

第一版简化：

```txt
格子只显示当前最后一次叠上的元素 + 总层数
```

例如一个格子依次叠：

```txt
水
火
土
火
```

最终显示：

```txt
火4
```

表示：

```txt
当前元素：火
总层数：4
```

# 11. 元素引爆

## 11.1 引爆伤害

元素层数越高，引爆伤害越高。

公式：

```txt
引爆伤害 = 1 + 2 + 3 + ... + 当前层数
```

示例：

```txt
1层 = 1
2层 = 1 + 2 = 3
3层 = 1 + 2 + 3 = 6
4层 = 1 + 2 + 3 + 4 = 10
5层 = 15
6层 = 21
```

## 11.2 引爆范围

默认十字 1 格范围：

```txt
  x
x O x
  x
```

```txt
O = 引爆中心
x = 受到爆炸影响的格子
```

## 11.3 点击预览

玩家点击元素格时，显示：

```txt
元素：火
层数：4
爆炸伤害：10
范围：十字1格
```

并在棋盘上高亮十字范围。

## 11.4 引爆后

```txt
元素格被清空
爆炸范围内敌人受到伤害
```

# 12. 元素块作为场上物

## 12.1 元素块独立存在

元素格不是纯视觉效果，它是独立场上对象。

```txt
元素块可以阻挡怪物移动
元素块可以被怪物攻击
元素块可以被引爆
```

## 12.2 怪物默认不攻击元素块

```txt
怪物默认目标是英雄
```

但如果元素块挡住了怪物路径：

```txt
怪物会攻击挡路元素块
```

## 12.3 怪物攻击元素块代价

```txt
怪物攻击元素块后，本回合所有行动点清空。
```

也就是说，元素块可以用来拖延怪物。

## 12.4 怪物触发引爆

如果怪物攻击了元素块，并触发元素块自身规则：

```txt
元素块会引爆
怪物也会受到爆炸伤害
```

这形成陷阱玩法：

```txt
玩家用元素块堵路
怪物被迫攻击元素块
怪物浪费行动
元素块爆炸反伤怪物
```

# 13. 怪物系统

## 13.1 第一版怪物简化

第一版基础怪物只做：

```txt
向英雄靠近
只装备向左攻击方块
攻击一次
伤害 = 怪物攻击力
```

怪物不使用玩家的复杂连段结算。

## 13.2 怪物移动目标

怪物从右上方出现，向左下方靠近英雄。

基础怪物目标：

```txt
移动到英雄右边
从右侧向左攻击英雄
```

示意：

```txt
H ← M
```

## 13.3 怪物攻击

```txt
怪物每次攻击只攻击一次
怪物攻击力是多少，就造成多少伤害
```

示例：

```txt
怪物攻击力 = 2
英雄受到 2 点伤害
```

## 13.4 怪物 AI 第一版

```txt
1. 如果能攻击英雄 → 攻击英雄
2. 如果不能攻击英雄 → 向英雄靠近
3. 如果被元素块挡住 → 攻击挡路元素块
4. 攻击元素块后，本回合结束
```

玩家回合期间，上述 AI 逻辑会实时作为**预览**在棋盘上展示：

```txt
怪物移动目标格：橙色高亮
怪物攻击目标格：红色高亮
目标英雄身上显示预计受伤数字
英雄移动后预览立即刷新
```

战斗采用交替回合：

```txt
玩家回合
怪物回合
玩家回合
怪物回合
```

```txt
第一波包含 5 个小回合
```

```txt
第4小回合：玩家行动 → 怪物行动
```

```txt
第一波结束
```


# 15. 第一场教学战斗

## 15.1 初始装备

玩家有两个元素使。

初始 3 个行动点：

```txt
行动点1：火·1格·1号·1阶，方向向右
行动点2：火·1格·1号·1阶，方向向右
行动点3：火·1格·1号·1阶，方向向右
```

或者初始给火、水两种：

```txt
行动点1：火·1格·1号·1阶
行动点2：水·1格·1号·1阶
行动点3：火·1格·1号·1阶
```

第一版建议用火、水，符合开局只有两个元素。

## 15.2 第一只怪

```txt
怪物血量：6
怪物元素：无
```

玩家三次基础攻击都命中：

```txt
第1次：1点
第2次：2点
第3次：3点
总伤害：6点
```

结果：

```txt
6血怪正好死亡
```

教学目标：

```txt
让玩家理解 3 个行动点 = 1 + 2 + 3 = 6
```

## 15.3 第二只怪与元素爆炸教学

场上出现第二只怪，并在附近放一个 10 血目标。

目标：

```txt
让玩家通过最后一次元素爆炸打出 9 点左右伤害
让 10 血目标剩 1 点
突出元素爆炸效果
```

示例结算：

```txt
第1段：1点
第2段：2点
第3段：3点 × 2 = 6点

总伤害：1 + 2 + 6 = 9点
```

10 血目标剩：

```txt
1血
```

玩家理解：

```txt
元素爆炸能明显提高伤害
如果叠得更高或补一个元素，就能击杀
```

# 16. 商店系统

## 16.1 商店出现时机

```txt
每波战斗结束后进入商店
```

第一版：

```txt
第一波 5 个小回合结束 → 进入商店
```

## 16.2 商店商品类型

商店主要卖两类东西：

```txt
1. 带元素的攻击方块
2. 随机元素
```

## 16.3 初始元素与商店元素

开局玩家只有：

```txt
火
水
```

但商店可以出现四种元素：

```txt
风
火
水
土
```

目的：

```txt
让玩家看到新颜色后产生购买欲望
引导玩家尝试新元素叠层和引爆
```

## 16.4 商店分类

沿用直白命名，方便开发测试：

```txt
直线攻击块商店
方块攻击块商店
L形攻击块商店
T形攻击块商店
十字攻击块商店
Z/S攻击块商店
阶梯攻击块商店
凹口攻击块商店
分叉攻击块商店
异形攻击块商店
```

## 16.5 商品展示规则

每个商店固定展示 3 个商品。

示例：

```txt
【十字攻击块商店】

1. 火·5格·25号·1阶
2. 水·5格·26号·1阶
3. 土·5格·27号·1阶
```

## 16.6 刷新规则

玩家点击刷新后：

```txt
当前 3 个商品消失
重新生成 3 个新商品
```

重点：

```txt
每次刷新只展示 3 个固定商品
```

## 16.7 额外随机元素

商店除了固定 3 个商品外，有概率额外生成：

```txt
0～3 个随机元素
```

示例：

```txt
固定商品：
1. 火·3格·7号·1阶
2. 水·4格·12号·1阶
3. 风·5格·25号·1阶

额外元素：
火元素
土元素
```

额外元素可以直接买来放到场上。

## 16.8 元素商品用途

```txt
直接放置到指定格子
给已有元素格补层
制造爆点
堵怪物路线
触发引爆
```

# 17. 合成升级

## 17.1 合成规则

两个相同攻击方块可以合成升级。

```txt
两个1阶 → 一个2阶
两个2阶 → 一个3阶
两个3阶 → 一个4阶
```

## 17.2 相同判断

第一版建议严格：

```txt
元素相同
形状编号相同
阶级相同
```

例如：

```txt
火·5格·25号·1阶
+
火·5格·25号·1阶
=
火·5格·25号·2阶
```

不能合成：

```txt
火·5格·25号 + 水·5格·25号
火·5格·25号 + 火·5格·26号
火·5格·25号·1阶 + 火·5格·25号·2阶
```

## 17.3 升级收益

```txt
1阶：×1
2阶：×2
3阶：×4
4阶：×8
```

成长特点：

```txt
指数爆炸
高阶方块非常强
```

# 18. 背包整理

## 18.1 背包建议尺寸

第一版建议：

```txt
8 × 8
```

原因：

```txt
1～6格物品能形成摆放压力
不会像6×6那么卡
也不会像10×10那么松
```

## 18.2 背包物品

背包里主要放：

```txt
攻击方块
元素道具
未来可扩展遗物
```

## 18.3 背包和战斗关系

玩家在商店购买攻击方块后，需要放入背包。

背包装不下时：

```txt
不能购买
或需要先整理 / 出售 / 合成
```

# 21. 第一版战斗流程样例

## 开局

```txt
两个元素使在左下。
怪物从右上出现。
玩家有3个基础攻击方块。
```

## 第1小回合

```txt
玩家拖动英雄到合适位置。
三个行动点都调成向右。
三次攻击命中6血怪。

伤害：
1 + 2 + 3 = 6

怪物死亡。
```

## 怪物回合

```txt
剩余怪物向英雄靠近。
如果走到英雄右侧，则向左攻击。
怪物只攻击一次。
攻击力是多少，就造成多少伤害。
```

## 第2～5小回合

```txt
玩家继续攻击、铺元素、调整方向。
怪物继续移动和攻击。
如果元素块挡住怪物，怪物攻击元素块，并失去本回合所有行动。
如果元素块被打爆，怪物受到爆炸伤害。
```

## 第5小回合结束

```txt
第一波结束。
进入商店。
```

## 商店

```txt
玩家购买带元素攻击方块。
可刷新商店。
可购买额外随机元素。
可合成两个相同方块升级。
```

# 22. 当前核心规则总结

```txt
游戏名：《元素背包史》

棋盘：13×13
英雄：2个元素使
行动点：每回合3个
方向：上、下、左、右可调
攻击：第1次1点，第2次2点，第3次3点
元素：风、火、水、土
显示：火4、水3、土2、风5
引爆：十字范围
引爆伤害：1+2+3+...+层数
怪物：第一版只会移动和向左攻击
波次：玩家回合与怪物回合交替，5个小回合一波
商店：每店3个商品，可刷新，额外0～3个随机元素
升级：两个相同方块合成，高阶伤害翻倍
商品名：火·5格·25号·1阶
```

这版 GDD 可以作为 v0.1。下一步最适合补一份 **SRD / 程序实现规则文档**，把数据结构、回合状态机、伤害结算顺序、商店刷新伪代码写出来。
