# 右侧宠物详情面板重设计提示语

主风格图：`web/assets/reference_main_style_battle_ui_2026-06-22.jpg`

用途：给美术、UI 落版或生图工具重设计战斗界面右侧宠物详情面板。只改右侧宠物详情信息层级与视觉样式，不改变三栏战斗结构，不改游戏规则。

## 代码数据来源

右侧宠物详情必须能承接当前代码字段：

- `src/uiAdapter.cjs`：`stripUnit()` 输出宠物字段，包含 `name/displayName`、`element`、`quality`、`role`、`hp/maxHp`、`atk`、`def`、`shield`、`ap/availableAp`、`moveRange`、`position`、`elements`、`mechanics/mechanicStatus`、`shape`、`slots`。
- `web/paper-battle.js`：`renderRight()` 当前描线版右栏，包含头像、名称、HP、攻击、AP、标签、三枚行动槽、当前形状说明、方向/AP/移动/施放控制。
- `web/ux-app.js`：`renderUnitDetail()` 当前丰富详情，包含技能说明、当前状态、面向方向、行动形状、机制、受击预警、单位元素层、脚下元素层、行动槽列表。

## 生图/设计主提示语

```text
请按参考图 `web/assets/reference_main_style_battle_ui_2026-06-22.jpg` 的整体风格，重新设计《元素背包史》战斗界面右侧宠物详情面板。

画面是 16:9 横版桌面棋盘战斗 UI，整体是暖木桌面 + 羊皮纸卡片 + 手绘描边 + 轻仙侠 Q 版宠物游戏风格。右侧面板固定在棋盘右边，宽度约占画面 20%-22%，高度从棋盘顶部延伸到按钮上方，材质为浅米色羊皮纸，边框有细木纹/烫金/卷角装饰，阴影柔和，不能做成现代 SaaS 卡片或深色科幻面板。

右侧面板展示“当前选中的我方宠物详情”，请以玉兔精作为示例，但所有文字都要像可替换 UI 文本，不要画死成插画标题。顶部是圆形或花窗式宠物头像，旁边显示宠物名、元素、品质、职业/定位；头像风格要和棋盘上的 Q 版角色一致，有白色描边和轻微立体贴纸感。

信息层级从上到下：
1. 宠物身份区：头像、名称、元素图标、品质徽章、角色定位。
2. 核心战斗数值：HP 当前/最大、攻击、防御、护盾、AP 当前/最大、移动范围，用小图标加短数字展示，必须一眼可扫，不要堆成长段文字。
3. 当前技能/行动形状：显示形状名、命中格数、元素层数、方向；用 5x5 小格预览十字形/T 形/菱形等作用范围，选中格用浅蓝描边，作用格用灰褐色块。
4. 三个行动槽：每个槽是一枚小纸片按钮，显示元素图标、方向箭头、是否已使用/可用；当前槽有蓝色发光描边，已用槽降低透明度。
5. 状态与机制：当前状态、面向方向、行动形状摘要、机制标签，用两行以内的标签/短句呈现。
6. 元素层：单位元素层、脚下元素层，使用火/水/风/土四色小徽章，数字可替换。
7. 受击预警：如果有预计伤害，在面板下部出现浅红警示条，显示预计伤害、HP 变化和存活/KO，不要遮挡主要信息。
8. 操作区：方向按钮、AP 选择、释放按钮保留在底部；主按钮是蓝色手绘按钮，和参考图“开始行动”按钮同风格，但尺寸更小。

视觉要求：
- 保持参考图的浅米、暖棕、淡金、浅蓝选中高亮；元素色只作为小徽章点缀。
- 面板内容要清楚分区，但不要用厚重卡片套卡片；用细分割线、浅底条、纸片按钮组织信息。
- 字体是清晰中文游戏 UI 字体，标题略粗，正文短而可读。
- 所有文本必须留出动态长度空间：宠物名 2-5 个中文、数值 1-3 位、槽名 2-6 个中文都不能溢出。
- 右侧面板不能压住棋盘、底部战报或“开始行动”主按钮。

禁止：
- 不要现代玻璃拟态、霓虹科幻、深蓝紫渐变、欧美写实 RPG、厚重金属边框。
- 不要把代码字段之外的系统画进去，例如法力、经验条、装备栏、星座、背包货币。
- 不要把长说明写成整段文章；所有说明必须适合前端动态替换。
- 不要改变当前三栏布局，也不要把右侧详情变成弹窗。
```

## 前端落版字段约束

右侧详情落版时按这些字段绑定，不新增规则字段：

| 区块 | 字段 |
|---|---|
| 身份 | `displayName || name`、`element`、`quality`、`role` |
| 数值 | `hp/maxHp`、`atk`、`def`、`shield`、`availableAp/ap`、`moveRange` |
| 技能 | `shape.shapeName`、`shape.shapeClass`、`shape.hitCells`、`shape.slotCount`、`shape.skill` |
| 行动槽 | `slots[].label`、`slots[].element`、`slots[].layers`、`slots[].shapeName`、`slots[].direction`、`slots[].used`、`slots[].canUse` |
| 状态 | `alive`、当前 `selected.direction` 或选中槽 `direction`、`mechanicStatus` |
| 元素层 | `unit.elements` / `unit.elementLayers`、脚下格 `cell.elements` |
| 预警 | `teamRisk.damage`、`teamRisk.hpFrom`、`teamRisk.hpTo`、`teamRisk.lethal`、`teamRisk.threats` |

## 交付建议

- 生图先出右侧面板单独裁切版，再出整屏嵌入版。
- PSD/源文件要分层：面板底、头像框、身份区、数值区、技能区、槽按钮、元素徽章、预警条、底部操作按钮。
- 不要把示例文本栅格化到背景层；文字必须独立可替换。
