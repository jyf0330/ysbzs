# 棋盘战斗界面出版规格

## 1. 文档身份

本文是棋盘战斗界面的 UI/UX 出版规格，用于统一策划、UI、美术、前端和测试对界面边界的理解。

本文定义当前首版必须落地的界面边界、备战交互和自动流程按钮边界；更深的 AI 寻路策略、商店经济规则和后续美术细化拆到独立任务。

## 2. 第一版界面目标

把战斗界面收束成清晰的三栏战斗台：

```text
顶部：状态栏
左侧：上阵宠物与行动块
中间：8x8 战斗棋盘
右侧：详细信息与回合控制
底部：事件 / 战报
备战：覆盖主战斗区的阶段界面
商店 / 奖励：只在对应阶段展示
```

核心原则：

- 左侧是我方可操作区，固定展示上阵宠物和每只宠物的行动块。
- 棋盘是战场，只显示位置、单位、阵营、选中态、范围态和极短状态。
- 右侧是说明区，展示当前选中宠物、行动块或棋盘格的完整详情。
- 备战是阶段切换，不是普通小弹窗；大覆盖层用于明确“现在不是开战状态”。
- 自动流程必须有清楚的按钮边界、禁用态和失败提示，不能让玩家误以为仍可无条件托管。

## 3. 区域职责

### 3.1 顶部状态栏

顶部保留：

```text
元素背包史 / YSBZS
棋盘战斗交互重构版
阶段 / 时间 / 回合 / 金币 / 版本
我方英雄 HP / 敌方 Boss HP
```

要求：

- 保持低高度，不压缩棋盘。
- 状态块使用统一纸片风格。
- 我方血量偏蓝，敌方 Boss 偏红。

### 3.2 左侧：上阵宠物与行动块

左侧标题使用：

```text
上阵宠物与行动块
```

左侧职责：

- 显示最多 4 个上阵宠物。
- 每个宠物卡下固定显示 3 个行动块。
- 总共 12 个行动块必须可扫视、可点击、不拥挤。
- 点击宠物卡：选中宠物，并让右侧显示宠物详情。
- 点击行动块：选中行动块，并让右侧显示行动块详情与方向状态。

左侧不负责：

- 展开完整技能说明。
- 展开完整释放条件。
- 展开完整方向控制面板以外的长文本说明。

行动块默认只显示：

```text
元素 / 简短技能名 / 当前方向 / 可用态
```

### 3.3 中间：战斗棋盘

棋盘只显示：

- 行列坐标。
- 我方单位、敌方单位、Boss。
- 选中框。
- 可移动范围。
- 可攻击范围。
- 元素地块轻量提示。
- 极短战斗结果，如 `伤3`、`死`。

棋盘禁止显示：

- 完整属性。
- 技能说明。
- 释放条件。
- 长目标描述。
- 多行机制解释。

点击棋盘格后，右侧详细信息刷新为该格信息。

悬浮棋盘只给轻提示，不改变右侧主详情面板。行动块选中时，棋盘悬浮不能覆盖行动块详情。

### 3.4 右侧：详细信息与回合控制

右侧标题使用：

```text
详细信息
```

右侧根据当前选中对象变化：

- 选中宠物：显示宠物属性、状态、元素层、机制、行动块列表。
- 选中行动块：显示所属宠物、元素、方向、消耗、攻击形状、释放条件、效果说明。
- 选中棋盘格：显示位置、单位、地块、元素层、预览或威胁摘要。
- 无选中对象：显示“请选择宠物、行动块或棋盘格。”

回合控制可以保留在右侧下半区，但视觉层级低于详细信息。

### 3.5 备战界面

备战不是小弹窗，而是覆盖主战斗区的大界面。

设计目的：

- 告诉玩家当前处于配置阵容阶段。
- 明确此时不能进行战斗操作。
- 让玩家把注意力转到宠物上阵、下阵和顺序调整。

第一版必须支持完整备战拖拽和背包筛选：

- 拖动宠物到“上阵阵容”或“背包 / 备战席”，完成上阵 / 下阵。
- 筛选框按名称、元素、职能过滤背包与上阵列表。
- 点击“准备开始”后锁定阵容，关闭备战台并进入战斗。
- 战斗开始后备战入口禁用。

### 3.6 底部事件 / 战报

底部事件区使用纸张风格，不使用黑底。

要求：

- 宽度优先与棋盘主区域对齐。
- 默认显示事件 / 战报摘要。
- 回放和调试可以作为标签页，但不应抢占战斗主视线。

### 3.7 商店 / 奖励

战斗阶段不常驻显示商店和奖励操作。

这些按钮只在对应阶段或合法状态下显示 / 可用：

```text
生成奖励
进入商店
刷新商店
离开商店
```

## 4. 自动流程边界

本规格定义两个按钮的 UI 位置、文字、禁用态和当前首版策略边界：

```text
一键完整流程
我方全部出击
```

- `一键完整流程`：从合法阶段触发完整流程；玩家手动移动或施放后，本场禁用完整自动流程。
- `我方全部出击`：只在玩家回合可用，按左侧行动块顺序释放；不自动移动、不重新摆位。
- 自动流程必须通过 `/api/action` 和核心 reducer，不得由 UI 直接修改核心状态。
- 目标无效、AP 不足、路径失败等情况使用结构化事件和 toast 给出可读提示。

## 5. 第一版验收

P0 第一版通过标准：

1. 左侧标题改为“上阵宠物与行动块”。
2. 左侧显示 4 个上阵宠物。
3. 每个宠物下面固定显示 3 个行动块。
4. 总共 12 个行动块能放下，不拥挤。
5. 点击宠物，右侧显示宠物详细信息。
6. 点击行动块，右侧显示行动块详细信息。
7. 棋盘格内部不堆大量文字。
8. 点击棋盘格，右侧显示格子详情。
9. 棋盘悬浮只显示轻提示，不覆盖右侧详情。
10. 备战按钮 / 覆盖层表达“当前不是开战状态”。
11. 底部事件 / 战报改为纸张风格。
12. 商店 / 奖励不在战斗阶段常驻抢占主界面。
13. 备战台支持完整备战拖拽和背包筛选。
14. 手动移动或施放后，一键完整流程进入禁用态。

P1 / P2 见 `docs/UI_COMBAT_LAYOUT_TASKS.md`。
