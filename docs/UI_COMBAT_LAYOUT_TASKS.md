# 棋盘战斗界面任务拆分

## P0：界面边界与信息层级

目标：先把玩家眼前的战斗台边界改正确。

范围：

- 左侧改为“上阵宠物”，标题右侧显示人数和 `备战` 核心入口。
- 左侧只显示上阵宠物必要战斗信息：名称、元素、HP、AP。
- 左侧每个宠物卡固定显示 3 个行动块，总计 12 个。
- 行动块改为可点击的小块：元素图标、短技能名、方向箭头。
- 右侧改为“详细信息”与“当前行动槽”两段：上半详情，下半方向 / AP / 释放。
- 右下角只保留 `我方全部出击` 和 `一键完整流程` 两个自动按钮。
- 玩家主界面移除 `自动战斗`，避免和 `一键完整流程` 重复。
- 棋盘不承载详情，只保留短名、阵营、血条 / 小血量和极短状态。
- 棋盘旁短状态条不显示目标详情。
- 底部事件 / 战报改为纸张风格。
- 底部事件 / 战报对齐中间棋盘宽度，不横跨全界面。
- 战斗阶段隐藏或降权商店 / 奖励入口。

当前仍不做：

- 商店完整改版。
- 超出当前 reducer 能力的自定义上阵排序。
- toast 提示系统的最终位置重排。

验收：

- 浏览器首屏能看清左侧 4 宠 12 行动块。
- `备战` 按钮在“上阵宠物”标题行右侧，战斗开始后锁定。
- 点击宠物、行动块、棋盘格时右侧详情切换正确。
- 点击行动块后，右侧当前行动槽可以调整方向并释放。
- 棋盘格无长文本详情。
- 中间状态条不再显示目标详情。
- 右下角只有两个核心自动按钮。
- `node --test tests/unit/ui_combat_layout_contract.test.cjs tests/unit/architecture_round4.test.cjs` 通过。
- `npm run check:browser` 通过并产出真实浏览器截图证据。
- 浏览器 console 无新增 error。

## P1：备战阶段覆盖层

目标：让玩家明确当前是阵容配置阶段，不是战斗阶段。

范围：

- 点击“备战”覆盖主战斗区。
- 棋盘被覆盖，表达当前不是开战状态。
- 展示未上阵宠物、当前上阵阵容、详情区。
- 支持按钮式上阵、下阵。
- 支持完整备战拖拽：宠物可在“上阵阵容”和“背包 / 备战席”之间拖放。
- 支持背包筛选：按名称、元素、职能过滤。
- 点击“准备开始”后锁定阵容并回到战斗棋盘。
- 左侧不常驻显示“备战 / 上场”列表。

不做：

- 多背包左右移动。
- 没有核心命令支持前，不做 UI 假排序。

验收：

- 备战覆盖层出现后棋盘不可操作。
- 准备开始后关闭覆盖层。
- 战斗开始后备战入口禁用。

## P2：自动流程系统

目标：把两个自动按钮背后的规则做完整。

拆分任务：

- `一键完整流程`：调用核心完整流程，自动推进战斗、奖励、商店、当天结束。
- `我方全部出击`：按左侧行动块顺序释放，不重新找路、不重新摆位。
- 不再提供玩家主界面的 `自动战斗` 按钮。
- 手动操作锁定：玩家手动移动或释放后禁用完整自动流程。
- 失败回退：目标无效、AP 不足、路径失败时给出可读提示。

验收：

- 自动流程有清晰的结构化事件。
- 手动操作后禁用态与提示一致。
- 自动流程不能绕过核心 reducer / API 入口。

## P3：视觉与美术细化

目标：把可爱手绘纸张风格做精。

范围：

- 统一纸张底色、棕色手绘线条、浅色边框。
- 宠物头像、小皇冠、小星星等轻装饰。
- 按钮、行动块、详情面板统一材质。
- 移动、选中、施放、阶段切换做轻动效。

验收：

- 首屏不再像黑底调试工具。
- 棋盘仍是视觉中心。
- 动效不遮挡战斗判断。
