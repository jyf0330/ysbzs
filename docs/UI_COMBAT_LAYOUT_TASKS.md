# 棋盘战斗界面任务拆分

## P0：界面边界与信息层级

目标：先把玩家眼前的战斗台边界改正确。

范围：

- 左侧改为“上阵宠物与行动块”。
- 左侧每个宠物卡固定显示 3 个行动块，总计 12 个。
- 右侧改为“详细信息”，承载宠物、行动块、棋盘格详情。
- 棋盘不承载详情，只保留短状态。
- 底部事件 / 战报改为纸张风格。
- 战斗阶段隐藏或降权商店 / 奖励入口。

不做：

- 完整备战拖拽。
- 自动寻路 / 自动选目标。
- 自动流程失败回退。
- 商店完整改版。

验收：

- 浏览器首屏能看清左侧 4 宠 12 行动块。
- 点击宠物、行动块、棋盘格时右侧详情切换正确。
- 棋盘格无长文本详情。
- `node --test tests/ui_adapter.test.cjs` 通过。
- `npm run check:ui-connected` 通过。
- 浏览器 console 无新增 error。

## P1：备战阶段覆盖层

目标：让玩家明确当前是阵容配置阶段，不是战斗阶段。

范围：

- 点击“备战”覆盖主战斗区。
- 展示未上阵宠物、当前上阵阵容、详情区。
- 支持按钮式上阵、下阵、调整 1-4 号位。
- 点击“准备开始”后锁定阵容并回到战斗棋盘。

不做：

- 拖拽。
- 背包筛选。
- 多背包左右移动。

验收：

- 备战覆盖层出现后棋盘不可操作。
- 准备开始后关闭覆盖层。
- 战斗开始后备战入口禁用。

## P2：自动流程系统

目标：把两个自动按钮背后的规则做完整。

拆分任务：

- `一键完整流程`：自动找怪、移动、选行动块、攻击、推进回合。
- `我方全部出击`：按左侧行动块顺序释放，不重新找路、不重新摆位。
- 手动操作锁定：玩家手动移动或释放后禁用完整自动流程。
- 失败回退：目标无效、AP 不足、路径失败时给出可读提示。

验收：

- 自动流程有清晰的结构化事件。
- 手动操作后禁用态与提示一致。
- 自动流程不能绕过核心 reducer / API 入口。

## P3：视觉与美术细化

目标：把可爱手绘纸张风格做精。

范围：

- 统一纸张底色、棕色手绘线条、浅色边框。
- 宠物头像、小皇冠、小星星等轻装饰。
- 按钮、行动块、详情面板统一材质。
- 移动、选中、施放、阶段切换做轻动效。

验收：

- 首屏不再像黑底调试工具。
- 棋盘仍是视觉中心。
- 动效不遮挡战斗判断。
