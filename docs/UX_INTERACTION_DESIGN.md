## 1. 核心交互模型

```
选择英雄 → 选择目标格 → 选择行动槽 → 施放
   │            │              │
   ▼            ▼              ▼
  左面板       棋盘          右面板
  点卡片       点格子         点槽位+调方向→点⚔
```

**回合循环（标准一天）：**
```
初始 → 玩家回合（移动+施放）→ 怪物回合（AI）→ 回合结算 → 商店 → 下一天
```

**三栏布局：**
```
┌──────────┬──────────────────────┬──────────┐
│ 英雄面板  │      棋盘 (8×8)      │ 行动槽面板 │
│ (左)     │     (中)             │ (右)     │
│          │                      │          │
│ 4张英雄卡 │ 格子：单位/元素/地形  │ 12个槽位  │
│ 可选中    │ 点击=移动目标        │ 可选中+调方向│
└──────────┴──────────────────────┴──────────┘
```

## 2. 操作清单

### 2.1 英雄选择

| 操作 | 触发方式 | 行为效果 | 视觉反馈 |
|---|---|---|---|
| **选中英雄** | 点击左侧英雄卡片 | `SELECT_UNIT` → 服务端记录 selected.unitId；棋盘上该单位高亮 | 卡片边框高亮 `#c8a050`；棋盘对应格显示 sel |
| **取消选中** | 再次点击已选卡片 | `SELECT_UNIT null` → 清除选中状态 | 卡片/棋盘取消高亮 |
| **切换选中** | 点击另一张卡片 | 旧选中取消，新选中生效 | 高亮转移 |

**关键状态：** `state.selectedHero` = 当前选中的英雄 unitId

#### 美术建议
- 卡片选中态应有**发光描边** + **微动效**（轻微上浮 2px + 阴影加深）
- 棋盘上被选英雄格应有 **脉动光晕**（而非简单边框变色），帮助玩家快速定位
- 未选中时卡片也应显示英雄的**元素色条**，方便视觉扫描

### 2.2 棋盘移动

| 操作 | 触发方式 | 行为效果 | 视觉反馈 |
|---|---|---|---|
| **点击空格** | 已选英雄 → 点空格 | `SELECT_CELL` + `MOVE_HERO` → 英雄移动到目标格 | 英雄图标移动到新位置；日志显示"🚶 XX 移动到(r,c)" |
| **点击有单位的格** | 点敌方/友方格 | 仅 `SELECT_CELL`，不触发 MOVE_HERO（有单位阻挡） | 格子上方显示威胁/信息预览 |
| **点元素格** | 点有元素层的格 | `SELECT_CELL` → 调试台显示元素详情 | 元素面板展开（元素类型+层数） |

**移动规则：**
- 只能移动到空格（无单位）
- 移动范围：1 格（相邻 4 方向：上下左右）
- 不能移动到棋盘外
- 移动后英雄仍保持选中状态

#### 美术建议
- **移动预览：** 选中英雄后，用半透明色块标记可移动的格子（相邻 4 格，绿色调）
- **移动轨迹：** 英雄移动时应有 **滑动动画**（0.2s 缓出），而非瞬间跳转
- **阻挡提示：** 有单位的格子用红色调底色，鼠标悬停显示单位名/HP
- **元素交互：** 英雄经过有元素层的格子时，播放**拾取粒子效果**（对应元素颜色的小粒子飞向英雄）

### 2.3 行动槽施放

| 操作 | 触发方式 | 行为效果 | 视觉反馈 |
|---|---|---|---|
| **选中槽位** | 点击右侧槽位卡片 | `SELECT_SLOT` → 高亮槽位；服务端记录 selected.slotId | 槽位卡片高亮 `#c8a050`；右侧面板滚动到该槽 |
| **使用槽位** | 选中后点 ⚔ 按钮 | `USE_SLOT` → 服务端结算该槽的元素效果 | 棋盘播放元素动画；日志显示"⚔ X 使用行动槽 N" |
| **调方向** | （尚未接入 UI） | `SET_ACTION_DIRECTION` → 改变槽的释放方向 | 棋盘显示方向指示箭头 |

**槽位结构：**
- 每英雄 3 个槽（共 4 英雄 × 3 = 12 槽）
- 每个槽绑定一个元素类型（火/水/风/土）
- 使用时从英雄出发，沿指定方向释放元素效果

#### 美术建议
- 每个槽位用**对应元素的主题色**（火=#d4855e, 水=#5e95b5, 风=#6ea86c, 土=#b8844a）
- 槽位选中时应有**元素图标脉冲动画** + 右侧**方向指示器**（上下左右箭头）
- 施放时：
  - **火：** 火焰喷射效果（从英雄到目标格，橙色粒子流）
  - **水：** 波浪扩散效果（蓝色涟漪向外扩散）
  - **风：** 旋风效果（绿色旋转粒子）
  - **土：** 地震/裂地效果（褐色碎片飞溅）
- 施放后目标格显示**元素层叠加动画**（数字跳动）
- 槽位 CD 提示：使用后槽位变灰（半透明）0.5s

### 2.4 回合控制

| 操作 | 触发方式 | 行为效果 | 视觉反馈 |
|---|---|---|---|
| **开始战斗** | 初始化后点"结束回合" | `START_BATTLE` → 进入 player_turn 阶段 | 顶部阶段标签变化；日志显示"⚔️ 第X天..." |
| **结束回合** | player_turn 阶段点"结束回合" | `END_PLAYER_TURN` → 轮到怪物行动 | 按钮变灰；等待怪物 AI 执行 |
| **AI 战斗** | 点"AI 战斗"按钮 | `RUN_MONSTER_TURN` → 全自动执行当前回合 | 棋盘快速演示单位移动/攻击动画 |
| **查看战报** | 点"战报"按钮 | 展示 `GET /api/report?mode=player` 文本 | 底部弹出战报面板 |

**阶段流转：**
```
init → player_turn → [END_PLAYER_TURN] → round_end → [AI] → player_turn → ... → battle_end → shop → day_end → init
```

#### 美术建议
- **阶段切换：** 屏幕顶部应有**阶段横幅动画**（"⚔️ 玩家回合"从上方滑入，停留 1.5s 后淡出）
- **结束回合按钮：** 颜色从绿色（可操作）→ 灰色（已结束/等待中）
- **AI 执行中：** 棋盘显示"AI 思考中…"遮罩层 + 快速单位移动动画（加速 3×）
- **回合计数：** 棋盘角落显示当前回合/最大回合（R1/2），用罗马数字风格增强仪式感

### 2.5 商店

| 操作 | 触发方式 | 行为效果 | 视觉反馈 |
|---|---|---|---|
| **打开商店** | 点顶部"🛍 商店" | `ENTER_SHOP` → 服务端生成 6 个商品 | 弹出商店模态框 |
| **购买** | 点商品旁的"购买" | `BUY_OFFER offerId` → 扣金币 + 获得宠物 | 商品卡片渐出 + 金币数跳动 + 成功音效 |
| **刷新** | 点"刷新"按钮 | `ROLL_SHOP` → 重新生成 6 个商品 | 旧卡片向左滑出，新卡片从右滑入 |
| **冻结/解冻** | 点商品旁的"冻结/解冻" | `FREEZE_OFFER` / `UNFREEZE_OFFER` | 商品卡片显示 ❄️ 冻结标记 |
| **离开商店** | 点"离开商店" 或 ✕ | `EXIT_SHOP` → 返回战斗 | 模态框关闭 |

#### 美术建议
- 商店模态框使用**毛玻璃背景**（backdrop-filter: blur），聚焦注意力
- 每个商品显示：宠物图标（元素图标）、名称、价格、稀有度（颜色边框）
- 购买动画：金币飞向商品 → 商品闪烁 → 宠物图标进入背包
- 冻结：❄️ 冰晶覆盖动画，商品边框变蓝
- 刷新：**旧卡左出、新卡右入**（0.3s cubic-bezier），避免突兀跳变
- 余额不足时购买按钮变灰，鼠标悬停提示"金币不足"

### 2.6 特殊试炼

| 操作 | 触发方式 | 行为效果 | 视觉反馈 |
|---|---|---|---|
| **进入试炼** | 点底部"第7天火核心试炼" | `SETUP_DAY7_FIRE_TRIAL` → 切换到 Day7 火元素试炼场景 | 棋盘风格切换（红色调）；日志显示试炼说明 |

#### 美术建议
- 试炼场景使用**不同色调的棋盘**（火试炼=暗红色调）
- 进入动画：屏幕闪红 → 棋盘格子依次变色 → 新单位入场

## 3. 状态流转

```
                    ┌──────────┐
                    │   init   │  ← NEW_GAME / initGame()
                    └────┬─────┘
                         │ START_BATTLE
                    ┌────▼─────┐
              ┌─────│player_turn│ ← 玩家操作（移动+施放）
              │     └────┬─────┘
              │          │ END_PLAYER_TURN
              │     ┌────▼─────┐
              │     │round_end │ ← 怪物 AI 执行
              │     └────┬─────┘
              │          │ 还有波次？
              │     ┌────▼─────┐
              │     │player_turn│ ← 下一小回合
              │     └────┬─────┘
              │          │ 波次结束
              │     ┌────▼─────┐
              │     │battle_end│
              │     └────┬─────┘
              │          │ 进入商店
              │     ┌────▼─────┐
              │     │  shop    │ ← 购买/刷新/离开
              │     └────┬─────┘
              │          │ 下一天
              └──────────┘
```

## 4. 美术交互建议

### 4.1 视觉层级（Z-index 体系）

| 层级 | 内容 | 说明 |
|---|---|---|
| z-0 | 棋盘背景 | 木质/ parchment 纹理 |
| z-1 | 棋盘格 | 64 格，深色木纹 |
| z-2 | 格子内容（单位/元素） | 单位图标、元素层 |
| z-3 | 选中高亮 | sel 类光晕 |
| z-4 | 移动预览 | 半透明绿色覆盖 |
| z-5 | 方向指示器 | 箭头从英雄指向目标 |
| z-6 | 浮动文字/伤害数字 | 施放时弹出 |
| z-100 | 商店模态框 | 毛玻璃背景 |
| z-200 | 阶段横幅 | 全屏宽顶部栏 |

### 4.2 色彩系统

```
元素色：
  火 🔥  #d4855e  (暖橙)
  水 💧  #5e95b5  (冷蓝)
  风 🌬  #6ea86c  (草绿)
  土 🪨  #b8844a  (赭石)

UI 色：
  背景      #1a1210  (深棕)
  面板      #2a201c  (中棕)
  边框      #4a3a30  (浅棕)
  文字      #eae0d0  (米白)
  辅助文字  #a09080  (灰米)
  强调色    #c8a070  (金)
  选中色    #c8a050  (亮金)
```

### 4.3 交互原则

| 原则 | 说明 | 实现方式 |
|---|---|---|
| **即时反馈** | 每次点击 100ms 内给出视觉响应 | 按钮按下态 50ms → 高亮/动画 150ms |
| **状态可见** | 选中、移动范围、可操作槽位始终可见 | sel 高亮 + 半透明预览 + 颜色编码 |
| **防误触** | 关键操作需二次确认 | 施放槽位：先选槽 → 再点 ⚔ |
| **一致性** | 所有可交互元素共用一套视觉语言 | 选中金框、悬停亮色、禁用灰色 |
| **动效时效** | 动画 < 0.3s，AI 批量动画可 3× 加速 | css cubic-bezier(0.2, 0, 0, 1) |

### 4.4 棋盘子项建议

每个 60×60px 格子内包含：

```
┌──────────────┐
│ 元素标签(8px) │  ← 火2 水1（仅有关联时显示）
│              │
│   单位图标    │  ← 🧙 👾 🏰（22px emoji 或精灵图）
│    HP 数字   │  ← 9px 右下角
└──────────────┘
```

**格子状态视觉：**
- 空格：显示 `·` 半透明点（无单位+无元素）
- 有单位：图标 + HP
- 有元素：元素标签（左上角，按颜色排列）
- 可移动（选中英雄时）：绿色半透明覆盖层
- 有单位阻挡：红色调底色（hover 显示单位名）
- 选中：金色发光边框

### 4.5 响应式布局

当前设计固定 1280×720 viewport，按比例缩放。建议保持固定分辨率 + CSS scale 缩放适配不同屏幕：

```css
#app {
  width: 1280px;
  height: 720px;
  transform-origin: top left;
  /* 通过 JS 计算 scale = Math.min(innerWidth/1280, innerHeight/720) */
}
```

## 5. 动画时机表

| 事件 | 触发 | 动画时长 | 动画类型 |
|---|---|---|---|
| 英雄移动 | 点空格 | 0.25s | translate 滑动 + 缓出 |
| 槽位施放 | 点 ⚔ | 0.4s | 粒子飞向目标 + 目标闪烁 |
| 元素叠加 | 施放后 | 0.3s | 数字跳动 (+1 弹出) |
| 伤害数字 | 受到伤害 | 0.6s | 数字浮起 + 渐隐 |
| 单位死亡 | HP ≤ 0 | 0.4s | 缩放消失 + 碎片粒子 |
| 阶段切换 | 回合变化 | 1.5s | 横幅滑入 → 停留 → 淡出 |
| AI 行动 | AI 回合 | 0.15s/步 | 快速滑动（3× 加速） |
| 购买 | 点购买 | 0.3s | 金币飞向商品 → 卡片闪烁 |
| 刷新商店 | 点刷新 | 0.3s | 旧卡左出，新卡右入 |
| 冻结 | 点冻结 | 0.2s | ❄️ 冰晶覆盖 |
| 全屏切换 | 点全屏 | 0s | 浏览器 API |
