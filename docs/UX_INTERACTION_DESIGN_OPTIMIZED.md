# 元素背包史 · UX 交互设计说明（优化版）

> 面向对象：UI/美术、前端、测试、AI 代码执行助手。  
> 目标：把“玩家怎么点、界面怎么反馈、状态怎么变、前端怎么验收”写成一份可执行说明。  
> 适用范围：8×8 棋盘战斗、英雄/宠物选择、移动、行动槽施放、回合控制、商店、第 7 天火核心试炼。

---

## 0. 文档原则

1. **玩家操作优先**：所有交互都从玩家真实入口触发，不用内部函数代替最终验收。
2. **状态可见**：当前选中单位、目标格、行动槽、可移动格、可攻击范围必须始终可见。
3. **反馈即时**：点击后 100ms 内必须出现按下态、选中态、预览或禁用提示。
4. **表现不反推规则**：UI 只展示核心状态和派生预览，不在 UI 层自行计算战斗规则。
5. **日志可回放**：每次移动、施放、元素变化、伤害、死亡、回合切换都必须进入结构化事件或战报来源。

---

## 1. 核心交互模型

### 1.1 玩家主路径

```text
选择我方单位 → 选择目标格 / 预览格 → 选择行动槽 → 调整方向 → 施放 → 查看结果 → 结束回合
```

### 1.2 三栏布局

```text
┌──────────────┬────────────────────────────┬──────────────┐
│ 左侧：宠物行动区│ 中间：8×8 棋盘              │ 右侧：详细信息区│
│              │                            │              │
│ 我方4个单位   │ 单位层 / 地形层 / 元素层 / 信息层 │ 当前选中详情   │
│ 每单位3行动块 │ 点击格子选择、移动、查看详情     │ 条件/方向/说明 │
└──────────────┴────────────────────────────┴──────────────┘
```

> 当前出版规格以 `docs/UI_COMBAT_LAYOUT_PUBLICATION_SPEC.md` 为准：行动块常驻左侧宠物卡下方；右侧只负责当前选中对象的详细信息、条件、方向状态和说明。

### 1.3 标准回合循环

```text
init
 → player_turn      玩家移动、选槽、施放
 → enemy_turn       敌方移动、攻击、试炼召唤
 → round_settle     元素、陷阱、死亡、奖励条件结算
 → player_turn      下一回合
 → battle_end       胜负判定
 → shop             商店
 → next_day / init  下一天
```

> UI 文案可以显示为：准备阶段 → 我方回合 → 敌方回合 → 结算阶段 → 商店 → 下一天。

## 2. 全局选择状态

### 2.1 必备状态

| 状态 | 字段建议 | 来源 | 用途 |
|---|---|---|---|
| 当前选中单位 | `selected.unitId` | 点击左侧卡片 / 点击棋盘我方单位 | 决定移动、行动槽来源、预览中心 |
| 当前选中格 | `selected.cell` | 点击棋盘格 | 展示单位、元素、地形、可达性详情 |
| 当前选中槽 | `selected.slotId` | 点击左侧宠物卡下方行动块 | 决定施放的元素、形状、伤害、附魔 |
| 当前方向 | `selected.direction` | 点击方向按钮 / 快捷键 | 决定槽位释放方向 |
| 当前阶段 | `phase` | 战斗状态机 | 控制哪些按钮可用 |
| 当前预览 | `preview` | 核心层派生 | 显示移动范围、攻击范围、元素变化、预计伤害 |

### 2.2 选择优先级

```text
点击我方单位：选中单位，并刷新该单位移动/攻击预览
点击敌方单位：选中格，只展示敌方详情和威胁范围，不切换我方单位
点击空格：选中格；若有我方单位且格子合法，可触发移动
点击行动槽：选中槽，并显示槽位释放范围
点击方向：更新方向，并刷新攻击预览
点击施放：按 selected.unitId + selected.slotId + selected.direction 执行
```

### 2.3 取消规则

| 操作 | 结果 |
|---|---|
| 再点已选单位卡 | 取消选中单位、槽位、方向预览 |
| 点击棋盘空白但不可移动格 | 只切换选中格，不清除单位 |
| ESC / 右键 | 优先清除槽位 → 方向 → 选中格 → 选中单位 |
| 进入敌方回合 / 商店 | 清空战斗选择状态 |

## 3. 操作清单

### 3.1 选择单位

| 操作 | 触发 | 命令 | 状态变化 | UI 反馈 |
|---|---|---|---|---|
| 选中单位 | 点击左侧单位卡 / 棋盘我方单位 | `SELECT_UNIT` | `selected.unitId = unitId` | 卡片金色描边；棋盘单位脉动光晕；显示移动格 |
| 切换单位 | 点击另一我方单位 | `SELECT_UNIT` | 替换 `unitId` | 高亮转移；预览刷新 |
| 取消单位 | 再点已选单位 / ESC | `SELECT_UNIT null` | 清空选择 | 高亮、移动格、攻击预览消失 |

#### UI 要点

- 左侧卡片始终显示：名称、元素、HP、盾、行动力、状态标签。
- 当前不可行动单位显示灰度，但仍可点击查看详情。
- 选中单位在棋盘和卡片上必须同时高亮，避免玩家找不到单位。

---

### 3.2 棋盘格交互

| 操作 | 触发 | 命令 | 行为 | UI 反馈 |
|---|---|---|---|---|
| 查看格子 | 点击任意格 | `SELECT_CELL` | 左侧/浮层显示格子详情 | 格子边框高亮，显示单位/元素/地形信息 |
| 移动单位 | 已选单位 → 点击合法空格 | `MOVE_UNIT` / `MOVE_HERO` | 单位移动到目标格 | 0.2s 滑动；日志记录起点→终点 |
| 点敌方格 | 点击敌方单位 | `SELECT_CELL` | 展示敌方 HP/盾/攻击范围/身上元素 | 红色威胁边框；不自动移动 |
| 点元素格 | 点击有元素层格 | `SELECT_CELL` | 展示火/水/风/土层数和来源摘要 | 元素标签展开；显示可触发状态 |
| 点非法格 | 超出移动范围/被占位/不可通行 | 无或 `SELECT_CELL` | 不执行移动 | 抖动 80ms；提示原因 |

#### 移动规则展示

```text
绿色：可移动
红色：有单位阻挡 / 敌方威胁
金色：当前选中单位
蓝色/橙色/绿色元素纹理：当前格已有元素层
灰色：不可通行地形
```

#### 重要实现口径

- 棋盘至少分四层显示：单位层、地形层、元素层、信息层。
- 陷阱、火/水/风元素可以和单位共格；单位占位不能重叠。
- UI 信息层只显示预览，不参与规则结算。

---

### 3.3 行动槽选择与施放

| 操作 | 触发 | 命令 | 行为 | UI 反馈 |
|---|---|---|---|---|
| 选中槽 | 点击左侧行动块 | `SELECT_SLOT` | 记录当前槽位 | 行动块金框；右侧显示元素、形状、范围和条件 |
| 调方向 | 点击 ↑↓←→ / WASD | `SET_SLOT_DIR` | 更新释放方向 | 棋盘显示方向箭头和命中预览 |
| 施放槽 | 点击「施放」/ ⚔ | `USE_ACTION_SLOT` | 核心层结算攻击、元素、附魔、触发器 | 播放施放动画；更新日志和战报 |
| 槽不可用 | AP 不足 / 阶段错误 / 已用尽 | 无 | 禁止施放 | 按钮灰化；hover 显示原因 |

### 3.4 行动块信息

每个行动块常驻在左侧宠物卡下方。左侧只显示短信息，完整说明进入右侧详细信息区。

左侧行动块建议显示：

```text
元素图标 + 槽位名
状态：可用 / 已用 / AP不足 / 需先选单位
当前方向：↑↓←→
```

右侧详细信息区显示：

```text
形状：A1 / A2 / B1 / B2 ...
消耗：行动力 / 次数
效果：攻击、铺元素、附魔摘要
释放条件与目标合法性
```

### 3.5 施放反馈顺序

```text
1. 槽位按钮按下态
2. 棋盘显示释放路径 / 覆盖格
3. 单位播放施放动作
4. 目标格播放元素动画
5. 数字跳动：HP、盾、元素层、AP
6. 触发英雄领域 / 附魔 / 元素引爆
7. 死亡或状态变化动画
8. 结构化日志追加
```

> 动画可以分步播放，但核心状态应一次性由服务端/核心层返回，前端按事件顺序演出。

## 4. 回合控制

| 操作 | 触发 | 命令 | 行为 | UI 反馈 |
|---|---|---|---|---|
| 开始战斗 | 点击「开始战斗」 | `START_BATTLE` | 进入 `player_turn` | 顶部横幅：第 X 天 · 我方回合 |
| 结束回合 | 点击「结束回合」 | `END_PLAYER_TURN` | 进入敌方/结算流程 | 按钮灰化；清空选择；播放阶段横幅 |
| 执行敌方回合 | 自动或点「AI 战斗」 | `RUN_MONSTER_TURN` | 敌方移动、攻击、召唤 | 显示 AI 行动轨迹，速度 3× |
| 自动跑完整场 | 点击「自动战斗」 | `RUN_BATTLE` | 连续执行直到结束 | 日志持续滚动；可随时暂停 |
| 查看战报 | 点击「战报」 | `GET_REPORT` | 打开文字战报 | 底部/侧边抽屉展示玩家可读报告 |

| 阶段 | 可操作 | 禁用 |
|---|---|---|
| `init` | 开始战斗、查看阵容 | 移动、施放、结束回合 |

| `enemy_turn` | 查看详情、加速/暂停 | 移动、施放、买商店 |
| `round_settle` | 查看详情、日志 | 移动、施放 |
| `battle_end` | 查看战报、进入商店/下一天 | 移动、施放 |
| `shop` | 买、刷新、冻结、离开 | 战斗移动、施放 |

## 5. 商店交互

| 操作 | 命令 | 行为 | UI 反馈 |
|---|---|---|---|
| 打开商店 | `ENTER_SHOP` | 生成/展示商品 | 商店模态框出现，背景压暗 |
| 购买商品 | `BUY_OFFER` | 扣金币，获得单位/道具 | 金币数字跳动；商品卡淡出；背包闪光 |
| 刷新商品 | `ROLL_SHOP` | 未冻结商品刷新 | 旧卡左滑出，新卡右滑入 |
| 冻结商品 | `FREEZE_OFFER` | 商品保留到下次刷新 | 冰晶覆盖，冻结角标 |
| 解冻商品 | `UNFREEZE_OFFER` | 商品恢复可刷新 | 冰晶破碎动画 |
| 离开商店 | `EXIT_SHOP` | 回到下一天/战斗准备 | 模态框关闭，顶部阶段更新 |

### 5.1 商品卡字段

```text
宠物头像 / 元素图标
名称 + 品质
价格
定位标签：输出 / 控制 / 召唤 / 治疗 / 坦克
核心字段：HP / 攻 / 盾 / 行动
技能摘要
按钮：购买 / 冻结
```

### 5.2 商店禁用反馈

| 条件 | 反馈 |
|---|---|
| 金币不足 | 购买按钮灰化，hover 显示“金币不足：需要 X，当前 Y” |
| 已购买 | 卡片淡出或显示“已售出” |
| 已冻结 | 冻结按钮改成“解冻” |
| 非商店阶段 | 商店按钮灰化 |

| 元素 | 表现 |

## 8. 动效规范

| 事件 | 时长 | 表现 | 备注 |
|---|---:|---|---|
| 按钮点击 | 50-100ms | 按下缩放 / 亮度变化 | 保证即时反馈 |
| 单位选中 | 150ms | 上浮 2px + 金框 + 阴影 | 卡片和棋盘同步 |
| 单位移动 | 200-250ms | translate 滑动，缓出 | 不瞬移 |
| 槽位施放 | 300-400ms | 元素轨迹飞向目标格 | 不阻塞核心状态更新 |
| 元素叠加 | 250-300ms | `+1 火` 数字跳动 | 多层可合并显示 |
| 伤害数字 | 500-600ms | 数字上浮并渐隐 | HP/盾变化要同时可见 |
| 单位死亡 | 300-400ms | 缩小、破碎、淡出 | 死亡后格子状态刷新 |
| 阶段切换 | 1000-1500ms | 横幅滑入、停留、淡出 | 可跳过或加速 |
| AI 行动 | 100-150ms/步 | 快速滑动 | 默认 3× 加速 |
| 商店购买 | 300ms | 金币飞向商品，卡片闪光 | 数字跳动必须准确 |
| 商店刷新 | 300ms | 旧卡左出，新卡右入 | 冻结商品不动 |

```text
R1 结算：R5C4 火3→火0。
```

## 10. 结构化事件与回放要求

### 10.1 每个玩家动作至少记录

```json
{
  "id": "evt_xxx",
  "round": 1,
  "phase": "player_turn",
  "actorId": "pal_072",
  "action": "USE_ACTION_SLOT",
  "from": { "r": 5, "c": 2 },
  "targets": [{ "r": 5, "c": 4 }],
  "slotId": "slot_1",
  "direction": "right",
  "changes": [
    { "type": "element_add", "cell": { "r": 5, "c": 4 }, "element": "fire", "before": 1, "after": 3 },
    { "type": "damage", "targetId": "enemy_001", "hpBefore": 30, "hpAfter": 24, "amount": 6 }
  ]
}
```

### 10.2 回放最低要求

| 要求 | 说明 |
|---|---|
| 可按事件顺序重播 | 移动、攻击、元素变化、伤害、死亡顺序一致 |
| 可暂停/快进 | 支持 1×、2×、3×、跳到下一回合 |
| 可从战报定位事件 | 点击日志行能高亮相关格子/单位 |
| 可复现随机站位 | 随机种子或最终站位必须写入 trace |
| UI 不重算规则 | 回放读取事件变化，不重新跑战斗逻辑 |

## 11. 响应式与分辨率

### 11.1 默认画布

```css
#app {
  width: 1280px;
  height: 720px;
  transform-origin: top left;
}
```

### 11.2 缩放策略

```js
const scale = Math.min(window.innerWidth / 1280, window.innerHeight / 720);
```

## 12. 前端实现注意事项

1. 所有玩家操作统一从 `dispatchGameAction` 或公开 API 入口进入。
2. 前端只根据返回的 `state + preview + events` 渲染，不在 UI 里写独立规则。
3. `selected` 状态可以在前端保留，但必须能和服务端状态同步。
4. 预览必须由核心规则派生，尤其是元素触发、陷阱、领域、附魔、推拉。
5. 动画播放读取 `events`，不要用动画结果修改核心状态。
6. 商店、试炼、战斗共用同一套按钮禁用、选中态、日志规则。
7. 不要把第 7 天火核心试炼做成完全独立页面逻辑；它应接入原主流程，只是一个特殊场景配置。

## 13. 浏览器验收清单

### 13.1 玩家真实链路

| 编号 | 验收项 | 标准 |
|---|---|---|
| UX-01 | 点击单位卡 | 卡片和棋盘单位同步高亮，移动预览出现 |
| UX-02 | 点击合法空格移动 | 单位位置变化，移动动画出现，日志记录起点终点 |
| UX-03 | 点击非法格 | 不移动，出现禁用/错误提示 |
| UX-04 | 选槽 + 调方向 | 棋盘出现攻击范围和预计元素变化 |
| UX-05 | 点击施放 | HP、盾、元素层、日志、棋盘渲染全部更新 |
| UX-06 | 元素达到3层 | 显示触发状态，伤害数字和元素清零准确 |
| UX-07 | 结束回合 | 阶段切换，玩家操作按钮灰化，敌方行动开始 |
| UX-08 | 商店购买 | 金币扣除、商品状态、背包/阵容变化一致 |
| UX-09 | 商店冻结刷新 | 冻结商品保留，未冻结商品刷新 |
| UX-10 | 第7天试炼 | 敌方右上4×4随机站位、敌方英雄 R1C8、试炼规则提示都显示 |
| UX-11 | 战报 | 能按顺序读懂移动、攻击、元素、伤害、死亡、结算 |
| UX-12 | 回放 | 能按事件重播一整场，不靠 UI 重新计算规则 |

### 13.2 最终验收口径

必须覆盖：

```text
玩家操作入口 → 状态变化 → 结构化事件/日志变化 → 棋盘渲染更新 → 战报/回放可读
```

直接调用内部函数只能作为单元测试或诊断，不能作为最终验收。

## 14. 给 AI 执行助手的接入说明

本文件用于指导 UI/交互改造。执行时按以下优先级处理：

1. 先保证公开玩家操作链路完整：点击、选中、移动、选槽、施放、结束回合、商店。
2. 再保证状态和日志准确：HP、盾、AP、位置、元素层、触发器、死亡、胜负。
3. 再补视觉反馈：高亮、预览、禁用、动效、颜色、阶段横幅。
4. 最后补商店动画、试炼皮肤、回放高级交互。

不要机械执行，结合当前项目真实代码、目录结构、已有规则和上下文，吸收有用部分，丢弃无用、过重、重复或不适配的部分，最终以项目实际情况和执行者判断为准。
