# 波次表规则说明（2026-06-09）

本说明以当前交付表 `00_全数据联动总表_01-09_波次规则交付版_20260609.xlsx` 为准。

## 1. 这次表的核心变化

`03_monster_waves` 现在不再把“宠物ID”理解成单个宠物，而是改成：

```text
宠物池-数量
```

例如：

```text
1,2,3,4,5-2
```

意思是：

```text
从 pal_001、pal_002、pal_003、pal_004、pal_005 里面随机出 2 只怪。
```

数字只是编辑简写，真正的源 ID 仍然是宠物主表里的 `pal_XXX`。

## 2. 品质权重

新增/保留字段：

```text
品质权重
```

格式固定为：

```text
青铜,白银,黄金,钻石
```

例如：

```text
90,10,0,1
```

表示：

```text
青铜权重 90
白银权重 10
黄金权重 0
钻石权重 1
```

建议把它理解成“权重”，不是严格百分比，所以不要求合计 100。

## 3. 本行威胁计算

当前代码读取 CSV 时，`本行威胁` 会按下面规则实时计算；旧版单宠物行没有品质权重时仍保留旧表里的威胁值：

```text
候选池平均效果分 × 出怪数 × 品质期望倍率
```

其中：

```text
候选池平均效果分 = 候选池内所有宠物的效果分平均值
出怪数 = 宠物池-数量 最后一个 - 后面的数字
品质期望倍率 = 青铜概率×1.0 + 白银概率×1.5 + 黄金概率×2.0 + 钻石概率×2.5
```

品质概率由权重归一化得到。威胁分里的基础分优先读取 `02_monster_templates.csv` 的 `面板分`；该宠物没有怪物模板时，才回退到 `01_pets.csv` 的 `效果分`。

例如：

```text
宠物池-数量 = 1,2,3,4,5-2
品质权重 = 90,10,0,0
```

按当前压缩包里的宠物主表，pal_001~pal_005 的效果分是 77、57、120、64、82，平均效果分是 80，则：

```text
品质期望倍率 = 90/100×1.0 + 10/100×1.5 = 1.05
本行威胁 = 80 × 2 × 1.05 = 168
```

## 4. 怪物模板表现在的意义

当前规则下，波次表已经直接使用宠物池，不再通过独立 `monster_id` 找怪。

所以 `02_monster_templates` 的建议定位是：

```text
补充宠物作为敌方出现时的默认行为
补充 AI 类型
补充敌方倍率
补充是否复制体
补充是否允许进入随机池
补充敌方标签，例如小怪、精英、试炼复制体
```

它不应该再承担“波次表出什么怪”的主引用职责。

## 5. 这次交付里的保守处理

代码接入后的正式处理：

- 怪物模板表参与威胁分：优先用 `02_monster_templates.csv` 的 `面板分`，缺模板时回退宠物主表 `效果分`。
- 同一行随机出怪默认先不重复；如果 `出怪数 > 候选池数量`，候选池耗尽后会重新放回继续抽，避免运行时崩溃。
- 品质随机是每只怪独立随机。
- 支持数字短写，也正式支持范围短写：`1~5-2`、`1-5-2`、`pal_001~pal_005-2`。

## 6. 已定稿边界

1. `出怪数 > 候选池数量` 时，采用“候选池耗尽后放回继续抽”。
2. 品质按“每只怪独立随机”。
3. 怪物模板表参与威胁分，优先使用敌方模板 `面板分`。
4. 范围写法 `1~5-2` / `1-5-2` / `pal_001~pal_005-2` 作为正式表格格式保留。

## 7. 建议定稿规则

建议先定成：

```text
波次表 = 时间 + 候选宠物池 + 出怪数 + 品质权重
宠物主表 = 唯一数值来源
怪物模板表 = 敌方行为补充，不做主引用
威胁分 = 候选池平均战斗分 × 出怪数 × 品质期望倍率
```

其中候选池平均战斗分的来源顺序是：

```text
02_monster_templates.csv 面板分 → 01_pets.csv 效果分
```
