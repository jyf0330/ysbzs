# Archive 说明

本目录保存历史文档、旧工作流、旧规则、旧策划文件。

这些内容**只用于追溯**，不作为当前规则依据。

如果 archive 内容和当前文档冲突，以 `docs/00_AI_START_HERE.md` 和 `docs/01_CURRENT_GAME_SPEC.md` / `docs/02_CURRENT_WORKFLOW.md` 为准。

## 目录结构

| 目录 | 内容 |
|---|---|
| `workflow/` | 旧 AI 工作流规则、项目规则、上下文快照 |
| `decisions/` | 旧决策记录（已合并到 `06_DECISION_LOG.md`） |
| `old_specs/` | 旧游戏设计、程序开发、美术、测试文档 |
| `old_tasks/` | 旧任务卡 |
| `old_chats/` | 旧聊天记录整理 |
| `misc/` | 根目录历史文件、备份、旧截图 |

## 使用规则

- 新 AI 进入项目时，不要主动读取 archive。
- 只有在任务卡明确点名某个 archive 文件时，才作为背景参考。
- archive 中的文档应视为"历史意图"而非"当前事实"。
- 代码是事实基线，当前文档是意图基线，archive 仅作追溯。
