## 三、攻击形状：从 SD 编号改为 ID + 坐标

JSON 重新定义了6种攻击形状，**不再用 SD[1]-SD[20]**：

| JSON ID | JSON名 | 坐标 | 等价SD | 用途 |
|---|---|---|---|---|
| single_front | 前方单点 | `[[1,0]]` | SD[1] | 火苗灵 |
| jump_single_2 | 跳格点射 | `[[2,0]]` | 🆕 跳过第1格 | 滴滴灵 |
| two_target_range | 双格水泡 | `[[2,-1],[2,1]]` | 🆕 | 泡泡灵 |
| front_empty_cell | 前方铺元素 | `[[1,0]]` + preferEmpty | SD[1]变体 | 火种灵/泉泉灵 |
| front_sweep_3 | 前排三格 | `[[1,-1],[1,0],[1,1]]` | SD[4]变体 | 岩岩灵 |
| l_shape | 小拐弯 | `[[1,0],[2,0],[2,1]]` | SD[8] | 风风灵 |

> ⚠️ 这意味着攻击系统要从 SD 编号查找改为 JSON 坐标查找，或者兼容两套。

## 四、英雄定义：从 UNIT_DEFS 改为 JSON tiers 结构

**讨论中的结构：**
```js
fire_starter: {
  levels: { 1:{hp:20, slots:[{el,sn,dir,tier}]}, 2:{...}, 3:{...} }
}
```

**JSON 结构：**
```json
{
  "id": "flame_sprite",
  "tiers": {
    "bronze": { "hp":20, "attackShape":"single_front", "hitCount":3, "layersPerHit":1 },
    "silver": { "hp":30, "attackShape":"single_front", "hitCount":3, "layersPerHit":2 },
    "gold":   { "hp":40, "attackShape":"single_front", "hitCount":3, "layersPerHit":3 }
  }
}
```

**关键变化：**
- `slots:[{el,sn,dir,tier}]` → `hitCount` + `layersPerHit` + `attackShape`
- 不再有3个独立行动槽，改为统一的 `hitCount` 次攻击，每次叠 `layersPerHit` 层
- 元素类型由 `element` 字段决定（不写在槽里）
- 不再有 `dir`（方向），形状默认向敌方

## 五、召唤系统：重大扩展

JSON 里的召唤系统比讨论中丰富很多：

### 5a. 召唤物种类

| ID | 名称 | 来源 | 特性 |
|---|---|---|---|
| tiny_spirit | 小小灵 | 空格召唤 | HP6, ATK1, 普攻 |
| fire_spirit | 火火灵 | 火格召唤 | 攻击叠火1层 |
| water_spirit | 泡泡灵 | 水格召唤 | 攻击叠水1层 |
| wind_spirit | 风风灵 | 风格召唤 | 移动AP+1 |
| earth_spirit | 岩岩灵 | 土格召唤 | HP+3 |

### 5b. 召唤物HP公式

讨论中：火灵/水灵 HP 固定 9。
JSON：`hp = 6 + consumedLayers`。

| 消耗层数 | HP | ATK |
|---|---|---|
| 0（空格） | 6 | 1 |
| 1 | 7 | 1 |
| 2 | 8 | 1 |
| 3 | 9 | 1 |
| 4 | 10 | 2 |
| 5 | 11 | 2 |
| 6 | 12 | 3 |

> 消耗6层的元素格 → HP12 ATK3 的召唤物，更强。

### 5c. 分分灵的特殊被动

讨论中：分分灵 "待设计"。
JSON：分分灵修改召芽灵的召唤行为——**把1只大召唤物变成2-3只小召唤物**。

```
青铜：1只 → 2只，每只HP×0.5
白银：1只 → 2只，每只HP×0.75
黄金：1只 → 3只，每只HP×0.6
```

这是"人海召唤流"的关键英雄。

## 七、需要更新的

### 7a. 命名（全文件替换）

```
火苗使 → 火苗灵
水滴使 → 滴滴灵
召唤使 → 召芽灵
兽语使 → 绒语灵
灵爆使 → 爆爆灵
分灵使 → 分分灵
水水使 → 泡泡灵
```

### 7b. 攻击系统

- 需要支持 JSON 格式的 `attackShape` ID → 坐标解析
- 需要支持 `preferEmptyCell`（火种灵打空地铺元素）
- `hitCount` + `layersPerHit` 替代 `slots[3]`

### 7c. 召唤系统

- 按元素格类型生成不同召唤物
- HP = 6 + consumedLayers
- 元素灵有不同攻击效果（叠层/加AP/加HP）

### 7d. 商店池

- 从随机改为按 `shopPools` 固定池抽取

### 7e. 分分灵

- 新增"修改召芽灵召唤行为"的被动

### 7f. 新增英雄

- 火种灵（铺火）、泉泉灵（铺水）、岩岩灵（T2前排）、风风灵（Day3）

### 删除

- 十字使/微风使（不在JSON中）
- 均衡使（不在JSON中）
