# 二、第一波教学脚本 v0.1

## 第一波目标

第一波不是考验玩家，而是让玩家理解：

```txt
1. 两个元素使可以拖动
2. 三个行动点可以调方向
3. 三次基础攻击 = 1+2+3 = 6点
4. 没打中怪也能铺元素
5. 元素可以叠层
6. 元素块可以挡怪
7. 元素引爆可以造成范围伤害
8. 5个小回合后进入商店
```

---

# 初始状态

## 棋盘

```txt
13×13
```

## 我方

```txt
元素使A：左下方
元素使B：左下方
```

## 初始装备

```txt
行动点1：火·1格·1号·1阶，默认向右
行动点2：水·1格·1号·1阶，默认向右
行动点3：火·1格·1号·1阶，默认向右
```

## 初始怪物

```txt
怪物1：6血，无元素，基础怪
怪物2：10血，无元素，教学目标
```

## 怪物规则

```txt
怪物从右上方靠近。
怪物只会向左攻击。
怪物每次只攻击一次。
伤害 = 怪物攻击力。
```

# 第1小回合：教学直接攻击

## 目标

让玩家理解：

```txt
3次攻击 = 1+2+3 = 6点
```

## 场景

```txt
怪物1：6血
怪物1站在玩家可以打到的位置
```

## 玩家操作

```txt
1. 拖动元素使A到合适位置
2. 三个行动点都设置向右
3. 依次释放行动点1、2、3
```

## 结算

```txt
第1次命中：1点
第2次命中：2点
第3次命中：3点

总伤害：6点
怪物1死亡
```

## 屏幕提示

```txt
三次攻击正好造成 1+2+3=6 点伤害。
```

# 第2小回合：教学没打中也能铺元素

## 目标

让玩家理解：

```txt
攻击不到怪物也不会浪费，会在攻击范围生成元素。
```

## 场景

```txt
怪物2距离较远，当前攻击不到。
```

## 玩家操作

```txt
1. 玩家调整一个行动点方向
2. 攻击空地
```

## 结算

如果火攻击打到空格：

```txt
空格生成：火1
```

如果水攻击打到空格：

```txt
空格生成：水1
```

## 屏幕提示

```txt
没有命中怪物时，会在攻击范围铺下元素。
```

# 第3小回合：教学元素叠层

## 目标

让玩家理解：

```txt
同一个格子可以多次叠元素。
层数越高，爆炸越痛。
```

## 场景

上一回合已经有：

```txt
火1
```

## 玩家操作

继续把攻击打到同一个格子。

## 结算示例

```txt
火1 → 水2 → 火3
```

显示：

```txt
火3
```

含义：

```txt
当前元素：火
总层数：3
爆炸伤害：1+2+3=6
```

## 屏幕提示

```txt
元素层数越高，引爆伤害越高。
```

# 第4小回合：教学元素块挡怪

## 目标

让玩家理解：

```txt
元素块是场上物，可以挡住怪物。
怪物攻击元素块后，会浪费本回合行动。
```

## 场景

怪物向左下靠近英雄。

玩家在怪物路径上铺一个元素块：

```txt
水3
```

## 怪物回合结算

怪物想移动，但被水3挡住。

```txt
怪物攻击水3
怪物本回合结束
```

如果第一版设定怪物攻击元素块直接触发引爆：

```txt
水3引爆
爆炸伤害：6
怪物受到6点伤害
```

## 屏幕提示

```txt
元素块可以堵路。
怪物打掉元素块后，本回合不能继续行动。
```

# 第5小回合：教学元素爆炸

## 目标

让玩家理解：

```txt
叠高元素后，引爆可以范围清怪。
```

## 场景

场上制造一个：

```txt
火4
```

旁边有 10 血目标。

## 玩家点击火4

显示：

```txt
元素：火
层数：4
爆炸伤害：10
范围：十字1格
```

范围：

```txt
  x
x 火4 x
  x
```

## 结算

如果直接用完整火4爆炸：

```txt
10血目标受到10点伤害
目标死亡
```

如果要保留“剩1血教学效果”，则设置本次教学爆炸为：

```txt
第1段：1
第2段：2
第3段：3×2=6

总伤害：9
10血目标剩1血
```

## 推荐采用

第一版推荐直接用清楚版本：

```txt
火4 = 10点
10血目标被炸死
```

如果你想强调“爆炸差一点就够”，再用 9 点版本。

## 第一波结束

第5小回合结束后：

```txt
进入商店
```
