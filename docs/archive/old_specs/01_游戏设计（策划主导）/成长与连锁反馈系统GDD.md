## 1. 系统目的

S1 已证明「水+召唤引擎」**逻辑链路**跑得通，但 deep-interview 的两条爽感验收 **尚未达成**：

- **AC1**：一局内出现「引擎成型临界点」，链每回合自我放大。
- **AC2**：功率曲线「难 → 临界 → 碾压」，临界后清怪跳升、点「下一步」续航。

本系统负责把「链路通」变成「**看得见、感觉得到的滚雪球与高潮**」，由两块组成：

1. **永久成长（局内）**——让引擎「越转越强」可量化、可感知。
2. **连锁反馈仪式化**——让每次结算的克制/引爆/清场有明确高潮反馈。

> 注意：本系统不引入战斗内随机（反支柱）。成长与反馈都建立在**全量可预判**之上。

## 3. 永久成长（局内）

### 3.1 设计意图

引擎「越转越强」目前只体现在 `healSummon → atk+1`。需要把**英雄/单位本身**也随引擎运转而成长，形成正反馈：召唤越多 / 治疗越多 → 角色更强 → 召唤更猛。

### 3.2 成长计数源（已存在）

`G.engineStats`：

| 键 | 现状 | 成长用途 |
|---|---|---|
| `summonCount` | ✅ 已统计 | 召唤里程碑 |
| `healCount` | ✅ 已统计 | 治疗里程碑 |
| `chainCount`（新增） | ❌ | 连锁里程碑（见 §4） |

### 3.3 成长规则（提案，数值占位可调）

**A. 召唤师阶位（召芽灵 / 召唤系）**

| 里程碑（summonCount） | 效果 |
|---|---|
| 每 3 次召唤 | 之后新召唤物基础 ATK +1 |
| 每 5 次召唤 | 召唤槽该回合可多召 1 格（上限合理） |

**B. 治疗师阶位（泉泉灵 / 绒语灵）**

| 里程碑（healCount） | 效果 |
|---|---|
| 每 4 次治疗 | `healSummons` 单次治疗量 +1 |
| 每 8 次治疗 | 治疗同时 +1 ATK 的基础上额外 +1（成长加速） |

**C. 局内持久**：成长在「同一局 run」内保留（跨小回合/跨波）；**不跨局**（跨局成长属 S4 meta）。

### 3.4 数据结构（提案）

```
G.growth = {
  summonTier: 0,   // 由 summonCount 派生的阶位
  healTier: 0,
  chainTier: 0,
}
```

- 在 `spawnSummon` / `healSummon` / 连锁结算后重算阶位，写 `G.growth`。
- 召唤/治疗数值函数（`calcSproutSpawnParams`、`healSummon`）读取 `G.growth` 叠加。

### 3.5 可观测

- HUD `#es` 引擎统计区追加：`成长 召T{n}/疗T{n}/连T{n}`。
- 阶位提升时 `glog` + 飘字「⚙ 召唤师阶位 ↑」。

## 4. 连锁反馈仪式化

### 4.1 现状（已部分实现）

`settleExplosions()` 已：

- 怪物格单体结算、空格阈值十字引爆。
- 克制时 `glog('⚡ 元素克制 ×2！')`、引爆时 `glog('💥 ...引爆...')`。
- `G.relics(old_fuse)` 给空格爆炸 +1（遗物加成钩子已在）。

**缺**：把这些零散 glog 聚合成「一次结算 = 一段连锁高潮」的结构化反馈。

### 4.2 连锁的定义

一次 `settleExplosions()` 调用内：

- **连锁段数 chainSegments**：发生引爆/克制结算的格子数（或引爆触发的二次波及次数）。
- **克制次数 advHits**：本次结算触发 ×2 克制的次数。
- **总伤害 totalDamage**：本次结算对怪物/敌方城堡造成的总伤害。

### 4.3 反馈产出（写入 `previewEvents` / 新 `settleReport`）

结算结束输出一个结构：

```
G.lastSettle = {
  chainSegments, advHits, totalDamage,
  killedCount, clearedWave: bool, perfect: bool
}
```

- HUD/飘字读取 `G.lastSettle`，弹「连锁 ×{chainSegments}！克制 ×{advHits}！合计 −{totalDamage}」。
- 纯逻辑/测试环境只需 `G.lastSettle` 数据正确，不依赖动画。

### 4.4 完美回合

- **判定**：一次结算（或一个玩家回合的结算合计）令**当前波全部存活怪物清空** → `perfect = true`。
- **奖励**：额外金币（占位：+3）；`glog('🌟 完美回合！连锁清场 +3 金')`。
- 计入 `engineStats.chainCount` / `perfectCount`（新增），喂养 §3 成长与未来 meta。

## 5. 与 A4 残留风险的关系（重要）

deep-interview 残留风险 **A4**：临界后全自动碾压会稀释 ITB 手动预判乐趣。

本系统对 A4 的态度：

- **成长**只放大「玩家主动运转引擎」的回报，不替玩家做决策。
- **连锁/完美回合**是对**主动摆位/落克制**的奖励，鼓励玩家继续手操而非纯点「下一步」。
- 若 S2 playtest 仍显示「临界后无脑续航」，则在 S3 引入难度回升 / 决策点（记录在 ROADMAP 待决 §6），**不在本系统内强行加随机**。

## 7. 明确不做（本 GDD 范围外）

- 战斗内随机（暴击/闪避/随机命中）——反支柱。
- 跨局 meta 永久解锁（S4）。
- 遗物体系扩张（S3 商店/遗物 GDD）。
- 连锁动画细节（属表现层；本 GDD 只定数据与触发）。
