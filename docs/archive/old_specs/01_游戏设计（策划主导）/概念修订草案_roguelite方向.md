## 0. 本轮方向决策（已拍板）

| 议题 | 选择 |
|---|---|
| 根基定位 | **B Roguelite**（杀戮尖塔式：强随机商店、跨局 meta、build 多样） |
| 城堡/守护目标 | **前置**：保护己方城堡 / 打掉敌方城堡，决策变为「清怪 vs 拦截」 |
| 优先强化的爽点 | **连锁引爆大爆发**（连锁/克制飘字、完美回合） |

核心纪律：**随机只在战斗外（商店/路径/遗物），战斗内永远零隐藏随机、全量可预判。**

## 1. 一句话定位（差异化）

> 杀戮尖塔的 roguelite 结构 ×（战斗内）Into the Breach 的确定性全预判 × 元素叠层连锁引爆，每局守住城堡、追求一次更大的连锁。

护城河（独有核心机制）：

- **元素格三位一体**：每层元素同时是「伤害源 + 阻挡地形 + 连锁燃料」。
- **循环克制即时连锁**：风→火→水→土→风，后者引爆前者，按被引爆层累计伤害结算。
- **战斗内全确定**：行动前可预判元素层、伤害、引爆范围、怪物反扑、谁会死。

---

## 2. 核心循环（城堡前置 + roguelite 嵌套）

```
30秒（招牌决策）：同色憋大 vs 落克制即时连锁 ┆ 这格打伤害 vs 砌墙拦扑城堡的怪
   ↓
5分钟（一场战斗）：守住己方城堡 N 回合 / 打掉敌方城堡 → 金币 + 奖励
   ↓
一局 run：地图路径推进（普通战 / 精英 / 商店 / 事件 / Boss）→ 构筑成长 → 通关或团灭
   ↓
跨局 meta：解锁新单位 / 元素 / 遗物，开放 Ascension 难度层
```

- **城堡 HP = 整局 run 的血条，跨战斗保留**。怪突破防线扣城堡 HP。
- 同一机制同时实现 ITB 的「守护张力」与杀戮尖塔的「跨战斗血量管理」。

## 3. 玩家决策（双层）

- **战斗内招牌双轴**
  1. 同色憋大（5 层火 = 15 伤）vs 落克制元素触发即时连锁。
  2. 这格用来打伤害 vs 砌成元素墙拦截扑向城堡的怪。
- **战斗外 roguelite 决策**：选路径（要钱 / 要精英奖励 / 要补给）、商店买单位还是遗物、合成方向、是否赌精英。

⚠️ 注意：6 行动槽 × 形状 × 方向 × 元素对「轻度策略」受众可能决策过载；靠全量预览扛认知负担，但需控制同屏可选项数量。

## 5. 爽点设计（重点：连锁）

- **连锁反馈仪式化**：结算时弹「连锁 ×N！克制 ×2！」飘字 + 伤害总结飘字 + 短震动 / 慢镜。
- **「完美回合」判定**：一次结算清空整波 → 额外连锁奖励金币。
- **成长与爽点绑死**：遗物围绕「放大连锁」设计（例：连锁每多一段 +X 伤害 / 连锁额外波及斜向 / 引爆不清空首层）。roguelite 变强直接喂养连锁爽点，而非纯堆数值。
- **Boss 战** = 大量怪同屏，是终极连锁舞台。

## 6. 设计支柱（pillars）与反支柱

支柱：

1. **算得清才打得爽**：战斗内零隐藏随机，全量预判。
2. **元素即一切**：每层元素同时是伤害、地形、连锁燃料。
3. **连锁是高潮**：一切成长最终服务于「做出更大的连锁」。
4. **守城即节奏**：城堡 HP 把每场战斗串成一条 roguelite 张力线。
5. **随机在局外**：随机只存在于商店 / 路径 / 遗物，绝不进入战斗结算。

反支柱：

- 不做战斗内暴击 / 闪避 / 随机命中（破坏预判）。
- 不做需要背板才能打的隐藏机制。
- 不做脱离连锁的纯数值堆叠 build。
