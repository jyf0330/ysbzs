## 1. 一句话定位

> **杀戮尖塔式 roguelite（局外随机）× 战斗内 Into the Breach 全预判 × 元素叠层连锁 × 守城堡；当前垂直切片先验证「水+召唤自动引擎」滚雪球。**

### 1.1 差异化（护城河）

- **元素格三位一体**：每层元素 = 伤害源 + 阻挡地形 + 连锁燃料。
- **循环克制连锁**：风→火→水→土→风；后者引爆前者，按被引爆层累计伤害。
- **战斗内零隐藏随机**：行动前可预判层数、伤害、引爆、怪物轨迹、谁会死。
- **物品 = 英雄（SAP）**：build 靠单位技能/被动联动，不是独立装备或遗物条。**遗物已砍掉——英雄的被动/主动就是「遗物」。**

### 1.2 设计支柱

1. 算得清才打得爽  
2. 元素即一切  
3. 连锁是高潮（宏观） / 召唤滚雪球是 v1 首验（微观）  
4. 守城即节奏  
5. 随机只在局外（商店 / 路径 / 遗物）

### 1.3 反支柱

- 不做战斗内暴击/闪避/随机命中  
- 不做背板隐藏机制  
- 不做脱离元素/连锁（或引擎链）的纯数值堆

## 3. 核心体验

### 3.1 宏观（完整游戏）

- **30 秒决策**：同色憋大 vs 落克制即时连锁；打伤害 vs 砌墙拦扑城堡的怪。  
- **5 分钟一场**：守己方城堡 / 打敌方城堡，金币与奖励。  
- **一局 run**：路径（战/精英/商店/事件/Boss）→ 构筑 → 通关或团灭。  
- **跨局 meta**：解锁单位/遗物/难度层。

### 3.2 当前垂直切片（v1 首验）

- 体验 **「引擎成型 → 越转越爽 → 点下一步续航」**（The Bazaar 式自动链 + SAP 召唤）。
- 详见 [`召唤引擎系统GDD.md`](召唤引擎系统GDD.md)。

## 4. 核心循环

### 4.1 战斗内（已实现骨架）

```
玩家回合（移动 + 行动槽/skill）
  → 元素结算 settleExplosions
  → 召唤物行动 runSummonActions（v1 新增）
  → 怪物回合 3AP
  → 小回合 / 阶段结束
```

### 4.2 局外（部分实现）

- **已实现**：按天/阶段商店（`SHOP_POOLS`）、单位购买与 2 合 1 合成、Day1 上午/中午/下午/夜晚。  
- **未实现**：分叉路径地图、跨战斗城堡 HP 条、遗物 build 主力、跨局 meta。
