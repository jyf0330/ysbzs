## 分类规则

- 现行规则写入 `docs/01_游戏设计（策划主导）/`、`docs/02_程序开发（程序主导）/`、`docs/03_美术资产（美术主导）/`、`docs/04_测试验收（测试主导）/`。
- 历史草稿进入 `docs/99_归档/`，并在文件头标明“仅作历史参考”。
- 新收到但未整理的外部资料可先进入 `data/inbox/`；整理后再迁入正式文档或 `docs/99_归档/`。
- 实现前不得直接引用根目录草稿或归档草稿作为规则源头。

## 迁移流程

1. 检查目标文件是否已存在，避免覆盖其他 AI 的工作。
2. 原文完整归档，保留来源。
3. 将可执行内容拆到正式文档，并标明来源与复核状态。
4. 更新 `docs/10_CHANGELOG.md`。
5. 用 `rg --files` 确认根目录不再残留同类草稿。
