# 实现接入说明

## 推荐接入顺序

1. 先读取 `01_pets.csv`，确认当前 127 宠主池。
2. 用 `04_mechanisms.csv` 建立机制ID注册表。
3. 用 `08_action_shapes.csv` 给每个宠物生成攻击槽。
4. 用 `11_英雄领域_联动版.csv` 在战斗开始时给双方英雄挂领域。
5. 用 `12_元素反应_联动版.csv` 替换旧的线性地形伤害逻辑。
6. 用 `13_第7天兽群试炼_联动版.csv` 初始化第7天试炼，不再在 `day7FireTrial.cjs` 中硬写完整战斗流程。

## 必须接入的核心逻辑

### 火爆

- 火层 >= 3 时形成爆火。
- 有敌方单位站在格上时立即引爆。
- 空格火3+保留为爆火陷阱。
- 敌方进入/被推入/被拉入时触发。
- 伤害 = 1+2+...+N。
- 引爆后火层清零。

### 水催化

- 有冲浪鸭领域时生效。
- 同格水层 > 0 且我方添加火/水/风层时，消耗1层水。
- 本次添加元素层数翻倍。
- 没有水核心时不毒、不回血。

### 风聚火

- 有疾风隼/风聚火领域时生效。
- 风系行动可把目标附近火层搬到目标格。
- 转移出来的火层可以参与后续火爆。

### 行动力

- 宠物都有总行动力。
- 我方宠物移动不消耗行动力。
- 敌方宠物移动消耗行动力。
- 攻击后本回合不能再移动。
- 行动3表示最多释放3个攻击槽。

## 第7天浏览器验收

必须通过玩家入口触发，不允许只调用内部函数作为最终验收。

验收链路：

`页面按钮 / dispatchGameAction -> reducer -> battle/core -> battleTrace -> DOM`

首回合验收文本必须包含：

- 水汽催化
- 火层翻倍
- 火爆 Σ(1..N) 伤害
- 风聚火 / 旋风收束
- 空格爆火陷阱
- 火层清零
- 骑士蜂黄金复制体死亡
- 精灵龙黄金复制体死亡
