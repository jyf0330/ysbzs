## 核心规则对照表

| 元素 | day7FireTrial.cjs 实现 | 数学公式 | 示例 |
|------|----------------------|---------|------|
| 🔥 **火引爆** | `fireDamage(N)` + `explodeIfEnemyOnFire` | Σ(1..N) = N(N+1)/2 | 火3层=6, 火4层=10, 火10层=55 |
| 💧 **水催化** | `consumeWaterCatalystForFire` | 消耗1水→火层×2 | 基础2层→4层 |
| 🌬 **风聚火** | `transferFire(from, to, amount)` | 搬运火层 | 火2→从第1行第7列到第2行第7列 |
| 🎯 **引爆条件** | 同格有单位 + 火≥3层 | 否则保持为陷阱 | 骑士蜂格火3→引爆 |
| 👾 **复制体** | 无元素、无铺场 | 标准战斗 | 4黄金原型 |

## 第1回合固定流程

```
移动阶段
  火绒狐: (3,1)→(2,4)   进入第3行第5列点火位
  冲浪鸭: (4,1)→(0,4)   进入第1行第5列水汽位
  融焰娘: (4,0)→(1,5)   进入第2行第6列爆心位
  旋风狸: (5,1)→(2,6)   站上第3行第7列火陷阱

攻击阶段（按顺序）
  火绒狐 ×3 → 骑士蜂         主格火+1，副格首击火+2/后续+1
  冲浪鸭 ×2 → 精灵龙         水汽催化水+1×2
  融焰娘 ×2 → 精灵龙         水催化→火×2，主格火+4/副格火+1，引爆
  旋风狸 ×1 → 骑士蜂         补刀，风击
  旋风狸 ×2 → 皮皮鸡         风击 + 聚火(第1/3行第7列→第2行第7列)
  旋风狸 ×3 → 皮皮鸡         继续压盾 + 聚火
  冲浪鸭 ×3 → (3,5)          铺下回合催化水

元素结算
  骑士蜂: 火3层→引爆6伤→死亡
  精灵龙: 火4层→引爆10伤→死亡  (水催化翻倍后)
  皮皮鸡: 火4层→引爆10伤→盾16/34HP  (未死，回合末回满盾)

敌方结束
  皮皮鸡: 护盾重生→20盾
  棉悠悠: 从(3,4)移动到(3,3)，AP 3→2
```

1. **第7天场景只实现了第1回合。** 第2~10回合是否也走固定脚本？还是需要通用化引擎？
