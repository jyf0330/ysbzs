## 1. 四层关系总表

| 层 | 管什么 | 是否参与结算 | 是否占格 | 实现位置 |
| --- | --- | :---: | :---: | --- |
| **单位层** | 英雄、怪物、召唤物、城堡 | 是 | 占单位位 | `board.js` monAt/heroAt 等 |
| **地形层** | 陷阱（从元素爆炸生成） | 是 | 不占单位位 | `G.terrainCells` + `elements.js` |
| **元素层** | 火/水/风/土层数、引爆判断、元素伤害 | 是 | 不占单位位 | `G.elementCells` + `elements.js` |
| **信息层** | 格子详情、预览伤害、日志、调试 | 否 | 不占格 | `ui.js` buildCellInfo / formatBattleLog |

## 2. 核心规则

### 单位层 vs 地形层

| 规则 | 说明 |
| --- | --- |
| 英雄 + 陷阱 | 可以同格，英雄默认不触发陷阱 |
| 怪物 + 陷阱 | 可以同格，怪物**每移动进入一格时立即触发**该格陷阱 |
| 召唤物 + 陷阱 | 按英雄侧处理，默认不触发 |
| 城堡 + 陷阱 | 第一阶段不混用，若存在只显示不触发 |

### 地形层 vs 元素层

| 规则 | 说明 |
| --- | --- |
| 空格元素引爆 → 生成陷阱 | `settleExplosions` 中空格引爆后，该格留下对应元素陷阱 |
| 陷阱叠加 | 已有陷阱 + 新陷阱 = 直接叠加，不覆盖 |
| 多元素陷阱共存 | 不同元素陷阱可共存，怪物踩入时一起结算 |
| 触发后清空 | 怪物触发陷阱后，该格陷阱清零（一次性） |

### 核心层 vs 信息层

| 原则 | 说明 |
| --- | --- |
| 核心层（可做） | 计算伤害、扣 HP、扣 AP、生成陷阱、写结构化日志事件 |
| 核心层（不可做） | 直接拼 UI 字符串、操作显示层数据 |
| 信息层（可做） | 读取状态、拼中文、显示颜色/图标/提示 |
| 信息层（不可做） | 修改 HP/AP/元素层数/陷阱层数 |

## 3. 数据结构

G.terrainCells:
```js
G.terrainCells = {
  "0,5": { fire: 3, water: 0, wind: 0, earth: 0 },
  "1,3": { fire: 0, water: 2, wind: 0, earth: 1 },
  // ...
};
```

Key-based map, same pattern as `G.elementCells`.

## 4. 核心函数清单

| 函数 | 文件 | 职责 |
| --- | --- | --- |
| `addTrapLayers(pos, el, layers)` | `elements.js` | 向指定格叠加元素陷阱 |
| `getTerrain(pos)` | `elements.js` | 读取该格地形 |
| `convertElementsToTrapsAfterExplosion(pos)` | `elements.js` | 元素引爆层数 → 地形陷阱 |
| `resolveTerrainOnEnter(monster, pos)` | `battle.js` | 怪物进入格时触发地形结算 |
| `buildTerrainInfo(pos)` | `ui.js` | UI 读取该格地形信息 |
| `buildCellInfo(state, pos)` | `ui.js` | 派生单格四层完整信息 |
| `formatBattleLog(event)` | `ui.js` | 结构化事件 → 中文日志 |
| `buildCellSummary(unit, elements, terrain)` | `ui.js` | 单格短文本概要 |

## 5. 数据流

```
行动槽 useSlot → G.elementCells 层数累加
    → settleExplosions → 空格引爆
        → convertElementsToTrapsAfterExplosion → G.terrainCells 陷阱叠加
        → 伤害结算（怪物格直接伤害/空格爆炸波及）
    → 怪物回合 monsterAct → 移动一格
        → resolveTerrainOnEnter → 踩陷阱扣 HP
        → 结构化日志事件 → ui.js formatBattleLog → 中文显示
```
