# UI_UX_START · UI/交互任务入口

UI 任务只关心：玩家看什么、点什么、如何理解战斗。

## 必读文件

1. `../00_AI_START_HERE.md`
2. `../01_CURRENT_GAME_SPEC.md`
3. `../04_CURRENT_UI_ART_SPEC.md`
4. `../05_ASSET_AND_FILE_INDEX.md`

## 当前 UI 重点

- 棋盘必须是主视觉。
- 战斗按钮必须明显。
- 右下角不要放反人类操作。
- 玩家需要清楚看到：
  - 英雄
  - 怪物
  - 当前行动槽
  - 目标格
  - 元素层数
  - 预计伤害
  - 怪物威胁
  - 战斗日志
- Debug 信息不能压住主棋盘。
- 始终显示玩家攻击预览和怪物行为预览（不依赖点击/悬停）。

## UI 任务原则

- 先保证可玩性，再做装饰。
- 先保证信息清楚，再做氛围。
- 不要让 Debug 面板抢游戏主体。
- 遵守 `04_CURRENT_UI_ART_SPEC.md` 中的配色和可读性基线。
- 涉及操作反馈需更新交互规范。
