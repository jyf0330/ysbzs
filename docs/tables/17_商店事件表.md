# 17 · 商店事件表（v1.0）

> 用途：商店中出现的随机事件，增加变化和决策深度
> 触发频率：每天最多 1-2 个事件
> 事件池：15-25 个

---

## 一、事件触发规则

| 规则 | 值 |
|------|:---:|
| 每日商店事件槽 | 午/晚各 1 个，概率触发 |
| 触发基础概率 | 40% |
| 前期(D1-3)概率 | 20%（教学简化） |
| 中期(D4-7)概率 | 50% |
| 后期(D8-10)概率 | 60% |
| 同事件冷却 | 2 天内不重复 |
| 事件不影响商品池 | 事件和商品独立出现 |

## 三、事件详细设计

### 3.1 E06 · 免费刷新
```
触发：商店打开时，概率 15%
效果：面板上「刷新」按钮今晚第一次免费
UI 表现：按钮闪烁 + "老板请客！" 文案
```

### 3.2 E07 · 流浪英雄
```
触发：商店打开时，概率 8%
效果：一个独立浮窗出现，展示 1 个 T1 英雄
玩家操作：选择「接受」→ 英雄自动归队
限制：D3 后出现，不占用商店格子
```

### 3.3 E11 · 赏金任务
```
触发：商店打开时，概率 10%
效果：提示「赏金任务！击杀精英 +5 金」
玩家操作：无选择，自动接受
下一战：额外生成 1 只精英
结算：胜利后金币+5，失败无惩罚
```

### 3.4 E15 · 福袋
```
触发：商店打开时，概率 10%
效果：花费 3 金购买随机福袋
产出：随机 T1(50%) / T2(30%) / T3(15%) / T4(5%)
UI 表现：抽奖动画，宝箱打开
```

### 3.5 E16 · 元素狂潮
```
触发：商店打开时，概率 10%
效果：下一战开局棋盘上随机 3 格带有 2 层随机元素
策略：玩家可以利用这些元素快速引爆或召唤
```

### 3.6 E17 · 合成祝福
```
触发：商店打开时，概率 5%（后期珍贵事件）
效果：本商店期限内，合成只需 1 个同名英雄（不消耗被合成的）
策略：关键合成时机投资
```

## 四、事件实装优先级

### P0（立即需要）
| 事件 | 原因 |
|------|------|
| E01 sale_random | 简单的折扣逻辑，增加商店决策 |  
| E06 free_refresh | 最容易实现，用已有刷新逻辑 |
| E09 free_gold | 最简单事件，直接给金币 |

### P1（Alpha 前需要）
| 事件 | 原因 |
|------|------|
| E04 sale_first_buy | 鼓励消费 |
| E02 sale_element | 需要元素筛选 |
| E11 challenge_bounty | 增加风险回报决策 |  
| E15 challenge_gamble_bag | 抽奖机制提升体验 |
| E16 special_element_overflow | 代码改动小 |

### P2（Beta 前需要）
剩余所有事件，为正式版增加丰富度。

## 五、集成方案

### 商店事件槽设计

```
┌─────────────────────────────┐
│  [英雄商品]  [英雄商品]  [英雄商品]  │  ← 常规商店  
│    [事件槽]  [刷新] [冻结]     │  ← 事件/操作区
└─────────────────────────────┘
```

事件槽显示：
- 无事件时：空白或提示文字
- 有事件时：事件卡片 + 按钮交互

### 代码集成点

| 组件 | 改动范围 | 预估工作量 |
|------|---------|:---------:|
| shop.js | 新增 `eventPool`、触发判断、事件执行 | 2-3 天 |
| ui.js | 新增事件浮窗、按钮、动画 | 1-2 天 |
| data.js | 事件定义表 | 0.5 天 |
| 测试 | 20-40 条事件测试用例 | 0.5 天 |
