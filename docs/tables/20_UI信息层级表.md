# 20 · UI 信息层级表（v1.0）

> 用途：定义所有信息点在 UI 上的位置、优先级和显示规则
> 目标：玩家每回合只需看最少信息做出最优决策
> 数量：50 个信息点

---

## 一、显示层级标准

| 层级 | 定义 | 颜色/样式 | 例子 |
|:----:|------|---------|------|
| L1 | 永远显示 | 高亮/粗体/大字号 | 英雄HP、城堡HP、回合数 |
| L2 | 悬停显示 | 次级 | 英雄详细属性 |
| L3 | 战斗中临时浮显 | 飘字/渐隐 | 伤害数字、叠层提示 |
| L4 | 可折叠/面板 | 面板切换 | 战斗日志 |
| L5 | 只在商店显示 | 交互控件 | 价格、标签筛选 |

## 三、信息展示策略

### 3.1 新手模式（前 3 天默认）

```
仅显示 P0 信息 + 引导提示
隐藏：战斗日志、详细被动、标签筛选
目标：玩家不会被信息淹没
```

### 3.2 正常模式（D4+ 默认）

```
P0 + P1 信息全部显示
P2 信息可选展开
战斗日志折叠(默认隐藏)
目标：完整信息但不过度
```

### 3.3 专家模式（设置切换）

```
P0 + P1 + P2 全部默认显示
额外：怪物移动路径、伤害预计算、元素分布分析
目标：精通玩家最大化信息量
```
