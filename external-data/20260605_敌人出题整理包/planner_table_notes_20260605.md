# 策划表整理说明 · 2026-06-05

## 本轮目标

本轮不是把运行时 YAML 直接摊平成大表，而是把三类策划资产整理成：

1. 主表好读
2. 复杂规则下沉
3. 后续程序可接

对应关系：

- `1_宠物总表_127_数值定位版_20260605.xlsx`
  - 给策划看定位、目标分、总强度、面板分、机制分
- `2_怪物波次表_Day1-Day5_20260605.xlsx`
  - 给策划看 Day1-Day5 前 5 回合的“出题”节奏
- `1_机制词条库_魔塔尖塔可计算版_20260605.xlsx`
  - 给策划看机制词条的题型、参数和可计算口径
- `planner_table_rules_20260605.yaml`
  - 放默认值、枚举、公式、离散档、字段边界
- `planner_table_notes_20260605.md`
  - 放字段解释、接入边界、验收口径

## 主表边界

### 宠物总表保留

- 编号 / 名称 / 元素 / 副属 / 品质 / 体型 / 定位
- 目标分 / 总强度 / 差值 / 面板分 / 机制分
- HP / 攻 / 防 / 盾 / 行动
- 机制 / 形状 / 范围 / 动作 / 技能 / 标签 / 可召 / 备注 / 内部ID

### 宠物总表不保留

- 价格 / 解锁日 / 池档 / 商店规则
- cells 坐标
- trigger / condition / effect / log
- fallback / legacy / debug 字段
- 720 行成长明细

这些内容要么去 YAML，要么去单独 MD，不再塞回主表。

## 数值口径

- 目标分来自 品质 × 体型
- 面板分 = `HP + 攻×3 + 防×3 + 盾×5 + 行动×5`
- 总强度 = `面板分 + 机制分`
- 差值 = `总强度 - 目标分`
- 机制分按 8 的倍数
- `HP` / `盾` 只用离散档
- 非钻石默认 `防=0`
- 行动默认 3，少量定位允许 2 / 4 / 5

## 怪物波次表口径

这张表只做 Day1-Day5 的战斗出题节奏，不兼做怪物模板总库。

每行是一组“本回合需要被玩家识别和处理的题”：

- 这组怪是什么题
- 希望玩家第几回合前处理
- 不处理会怎么罚
- 应该被什么类型的宠物解掉

不把完整 trigger 树、日志模板、调试字段塞到这张表里。

## 机制词条库口径

这张表不是灵感摘抄，而是“词条 -> 出题目标 -> 触发时机 -> 计算口径 -> 机制分 -> 克制关系”的索引表。

主表只保留：

- 题型
- 适用对象
- 触发时机
- 关键参数
- 计算口径
- 常用取值
- 机制分
- 克制解法
- 被克制
- 联动
- 首版优先级

其中：

- `适用对象` 只用 `敌人 / 宠物 / 双用`
- `触发时机` 只用 `开场 / 每回合 / 受击时 / 攻击时 / 死亡时 / 存活时 / 进入范围时`
- `机制分` 必须是 8 的倍数
- `首版优先级` 只用 `必做 / 可做 / 后置`

更细的参数 schema、拆分规则和打分带宽放 YAML。

### 本轮拆分硬规则

以下机制不能再混写：

- 伤害类：`无视防御 / 真实伤害 / 固定伤害`
- 坚固类：`坚固（阈值） / 坚固（刮痧）`
- 护盾类：`初始盾 / 每回合盾 / 护盾刷新 / 厚盾低血 / 护盾转攻`

### 机制分口径

- 普通轻机制：`8 / 16`
- 明确解法机制：`24 / 32`
- 强门槛机制：`40 / 48`
- 精英 / Boss 机制：`56 / 64+`

首版词条优先体现“能出题、能计算、能接入”，不追求一次把所有花样塞满。

## 接入建议

后续如果要程序接：

1. 先把宠物总表主字段映射到单独 YAML 源
2. 让 `shape/cells`、`trigger/effect`、`growth` 继续维持分文件
3. 怪物波次表继续做策划视图，不直接当运行时 schema
4. 机制词条库作为“设计参数字典”，不要直接混到战斗主表

## 验收清单

- 三张主表都保持单 sheet、单主题
- 主表字段短、中文化、可扫读
- YAML 承接复杂公式和默认值
- MD 解释字段职责和接线边界
- 不改运行时代码
- 不手改 generated-json
