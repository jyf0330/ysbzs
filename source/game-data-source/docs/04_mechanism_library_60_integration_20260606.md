# 04_机制词条库 · 60条策划好读版接入说明

生成时间：2026-06-06 10:40

## 结论

这版不是“当前三表已用机制覆盖版”，而是完整的第一版机制词条库：

- `M00 none`：正式无机制占位。
- `M01-M60`：60条正式策划机制。
- 表格只服务策划查看、筛选、复查。
- YAML 服务程序接入，放触发、条件、参数、效果、结算顺序、日志模板。
- 宠物表、怪物模板表、事件表、波次表只引用 `机制ID`，不要复制完整机制规则。

| 范围 | 分类 | 用途 |

| M41-M50 | 元素/陷阱 | 火、水、风、土、多元素、陷阱叠加、火魔十字 |

## 接入优先级

第一批建议优先接这些，能最快支撑现有战斗闭环：

1. `none`
2. `mech_shield_flat`
3. `mech_armor_flat`
4. `mech_death_explosion`
5. `mech_self_destruct`
6. `mech_grow_atk_each_round`
7. `mech_grow_shield_each_round`
8. `mech_castle_line_damage`
9. `mech_bonus_reward_under_round5`
10. `mech_penalty_after_round10`

## 不要急着接的高风险机制

这些机制强，但需要更多系统支撑：

| 机制ID | 原因 |
|---|---|
| `mech_damage_cap_per_round` | 会强行防秒杀，需要回合伤害累计 |
| `mech_second_phase` | 需要阶段切换与状态替换 |
| `mech_copy_weak_self` | 容易爆场，需要召唤上限/位置规则 |
| `mech_fire_cross_detonate_diamond` | 只属于钻石火魔，不允许全局启用 |
| `mech_earth_block_cell` | 需要地形层/单位层分离 |
| `mech_countdown_pressure` | 需要波次级回合事件钩子 |

## 运行时规则

- `none`：安全跳过。
- `可接入`：允许进入 runtime，但仍要有对应 handler。
- `待接入`：可被表格引用，但运行时必须显示 pending，不得静默生效。
- `禁止接入`：不得进入 runtime。
- 未知 `机制ID`：脚本必须报错或生成明确缺口报告。
- `REVIEW`：不是正式机制ID，不得进入 runtime。

## 已定边界

1. 机制表不负责写宠物/怪物具体数值，数值仍以宠物主表和怪物模板数值源为准。
2. 机制分只参与估算，不等于最终伤害。
3. 火魔十字引爆只在钻石火魔达成条件时启用。
4. 陷阱是地形层，可与单位层共存。
5. 同格多元素/多陷阱优先保留爽感，不提前削弱。
6. 5回合高奖、10回合失败是波次级硬规则，优先接。

## 验收规则

接入脚本需要检查：

- 机制ID唯一。
- 机制编号唯一。
- 宠物/怪物/事件/波次引用的机制ID都能在机制库找到。
- `pending_adapter` 机制不会静默生效。
- `none` 不生成任何效果。
- 日志模板存在。
- 参数默认值能被解析。
- 触发点在 `trigger_order` 中存在。
