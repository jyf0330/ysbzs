# 06_商店奖励池表 接入说明

## 结论

`06_商店奖励池表_策划好读版_20260606.xlsx` 是商店/奖励系统的人工管理入口。

它和宠物表的关系是：

```text
01_宠物主表.宠物ID
  ↓
06_商店奖励池表.宠物ID
```

商店表只引用 `宠物ID`，名称、元素、品质、定位、标签都以宠物表为准。表里显示这些字段，是为了策划看表方便，不是第二套真源。

## 表格字段怎么改

### 只建议手改

| 字段 | 说明 |
|---|---|
| 宠物ID | 商品引用的宠物，必须存在于宠物主表 |
| 商品类型 | 第一版默认宠物，后续可扩展遗物/事件奖励 |
| 商店状态 | 启用/禁用/测试 |
| 解锁日 | 第几天可进入商店/奖励池 |
| 池档 | pT1/pT2/pT3/pT4 |
| 价格覆盖 | 特殊价格才填，空着走 YAML 默认价 |
| 夜市权重 | 基础夜市抽取权重 |
| 元素店权重 | 元素定向商店抽取权重 |
| 定位店权重 | 定位定向商店抽取权重 |
| 品质店权重 | 品质商店抽取权重 |
| 奖励权重 | 战斗/事件奖励池抽取权重 |
| 出现条件 | 简短条件，复杂条件写 YAML |
| 备注 | 策划备注 |

### 不建议手改

| 字段 | 来源 |
|---|---|
| 名称(自动) | 宠物表 |
| 元素(自动) | 宠物表 |
| 品质(自动) | 宠物表 |
| 定位(自动) | 宠物表 |
| 标签(自动) | 宠物表 |
| 默认价 | YAML 价格规则 |
| 商店池(自动) | 元素/定位/池档自动生成 |
| 奖励池(自动) | 池档自动生成 |

## 商店类型

当前 YAML 定义 10 类商店：

1. 基础夜市
2. 元素定向商店
3. 定位定向商店
4. 品质商店
5. 折扣商店
6. 复制/合成商店
7. 事件商店
8. 战斗奖励池
9. 精英/高奖池
10. 特殊商人

## 接入脚本建议

生成 runtime 时建议输出：

```text
shop_pools.json
shop_price_rules.json
reward_pools.json
```

最小结构：

```json
{
  "night_base": [
    {"petId": "pal_001", "weight": 100, "unlockDay": 1, "price": 2}
  ],
  "elem_火": [],
  "role_输出": [],
  "tier_pT1": [],
  "reward_pT1": []
}
```

## 验收规则

脚本必须检查：

1. 商店表所有 `宠物ID` 都存在于宠物表。
2. `宠物ID` 不重复。
3. `池档` 只能是 `pT1/pT2/pT3/pT4`。
4. `商店状态` 只能是 `启用/禁用/测试`。
5. 权重必须是数字，空值按 0 处理。
6. `价格覆盖` 为空时使用 YAML 默认价。
7. 显示字段必须能从宠物表重建，不能当真源。

## 边界

- 不要把商店价格、权重、解锁日塞回宠物表。
- 不要把复杂事件条件塞进商店表。
- 不要重复维护宠物名称、元素、品质、定位。
- 宠物表改基础信息后，商店表显示字段应由脚本或公式重新生成。
