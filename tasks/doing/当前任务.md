## 任务类型

排查 / 修 bug / 验收：元素场持久化与占用同步

## 任务目标

修复 `elementCells` 与 `G.board` 分裂导致的元素地形丢失、英雄可走入隐形元素格、怪物攻击元素块后残留元素层问题。

## 完成状态

✅ 已完成（2026-05-26）

## 完成内容

1. 新增 `topElementAt(pos)` / `hasElementAt(pos)`，元素占用判断统一以 `elementCells` 为准，并兼容 `board.el/stk`
2. 新增 `syncBoardElementFromElementCells(pos)`，结算后把未消耗元素重新同步为棋盘地形，避免出现隐形元素
3. 新增 `clearElementAt(pos, el)`，清除元素时同步更新 `elementCells` 与 `board`
4. `settleExplosions()` 清除被结算元素后同步剩余元素到 `board`
5. `moveHero()` 和底层 `MOVE_HERO` action 都禁止进入任意元素格
6. `monsterAct()` / `simMonAct()` 改用统一元素占用判断，怪物攻击元素块时同步清除元素场
7. `placeEl()` 避免把商店元素放到 `elementCells` 仍占用的格子
8. 新增 K 组测试 3 条（case_k_011~013），覆盖多元素结算后剩余元素显形并阻挡英雄、怪物攻击元素块清场、底层 MOVE_HERO 不能绕过占用校验；测试基准 240 → 243
