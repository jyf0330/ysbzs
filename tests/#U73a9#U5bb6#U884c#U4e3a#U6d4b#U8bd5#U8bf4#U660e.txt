《元素背包史》玩家行为测试说明
版本：2026-06-07 full coverage text core
目标：验证新架构下，玩家从战斗开始到夜晚商店结束的完整行为链路都能通过纯文字核心跑通。
架构链路：SourceTables → NormalizedData → Command → Reducer → StateChange/EventLog → TextReport
UI 状态：不接 UI、不接 DOM，只验证玩家行为、数据变化、事件日志和文字输出。

1. 安装/运行方式

   - 新增 full coverage 行为测试：96/96 passed
   - 无 DOM/UI 扫描：PASS
   - full day 文字报告检查：PASS

二、玩家完整行为链路验收

P01 进入战斗
玩家看到：第1天上午战斗开始，显示本场目标：5回合内清场拿高奖，10回合内必须清完。
系统命令：RUN_BATTLE
系统事件：BATTLE_START
验收点：phase 进入 battle；事件日志有 BATTLE_START；文字报告出现“战斗开始”。
对应测试：T21、T24、T80、T81、T95

P02 敌方刷怪
玩家看到：敌方Boss/波次召唤怪物，怪物显示名称、HP、攻击、位置规则。
系统命令：内部 runRound → spawnWave
系统事件：ROUND_START、SPAWN_ENEMY
状态变化：state.units 增加 enemy 单位；单位引用 03_怪物波次 和 02_怪物模板。
验收点：1-10天上午/下午所有波次都能定位；134条波次都能按 day/period/round 找到。
对应测试：T05、T12、T13、T83、T84

P03 玩家选择行动槽
玩家看到：我方英雄列表、每个英雄3个行动槽；点击某英雄第N槽后显示形状、元素、目标。
系统命令：内部 runPlayerTurn
系统事件：PLAYER_TURN_START、PLAYER_SELECT_SLOT
状态变化：当前不会扣血，只记录玩家选槽和目标。
验收点：事件日志必须记录 actorId、slot、shapeId/shapeName、element、target。
对应测试：T14、T15、T95

P04 行动槽叠元素
玩家看到：命中目标后，目标身上的火/水/风/土层数变化，例如 火层0→1、火层1→2、火层2→3。
系统命令：applyElement
系统事件：APPLY_ELEMENT
状态变化：target.elements[element] 增加层数。
验收点：至少一次完整战斗中出现 APPLY_ELEMENT；文字报告能读到层数前后变化。
对应测试：T14、T57、T95

P05 玩家结束行动并统一结算
玩家看到：点击“结束行动/统一结算”后，所有元素层统一结算，不在每次点击时立即扣血。
系统事件：PLAYER_TURN_END、ELEMENT_SETTLE、DAMAGE、UNIT_DEAD
状态变化：元素层数转化为伤害；护盾和HP发生前后变化；死亡单位 alive=false。
验收点：DAMAGE 事件必须包含 raw、final、shieldFrom、shieldTo、hpFrom、hpTo；死亡必须产生 UNIT_DEAD。
对应测试：T14、T16、T17、T57、T93、T95

P06 怪物行动
玩家看到：如果玩家没有清场，敌方怪物锁定我方目标并攻击。
系统命令：runMonsterTurn
系统事件：MONSTER_INTENT、DAMAGE
状态变化：英雄 HP/盾发生变化。
验收点：有敌人和英雄时必须产生意图；没有英雄时安全结束。
对应测试：T18、T19

P07 战斗结束与奖励
玩家看到：胜利/失败结果、奖励评级、金币变化；失败时扣我方英雄防线和经济倍率。
系统事件：BATTLE_END、BATTLE_FAIL_PENALTY
状态变化：state.result 写入 code/grade/gold；state.gold 增加；失败扣 castleLine/economyMultiplier。
验收点：胜利、快速胜利、失败惩罚路径都能跑。
对应测试：T21、T22、T56、T80

P08 奖励候选与选择奖励
玩家看到：战斗后出现3个奖励候选，可以选宠物或遗物/祝福。
系统命令：REWARD_OPTIONS、PICK_REWARD
系统事件：REWARD_OPTIONS、REWARD_PICK
状态变化：奖励进入 inventory 或 relics。
验收点：reward_pT1/pT2/pT3 等奖励池可生成候选；越界选择返回 false 不崩溃。
对应测试：T75、T76、T79、T80

P09 进入夜晚商店
玩家看到：进入第N天夜晚商店，显示当前金币、商品池、6个商品位。
系统命令：ENTER_SHOP
系统事件：SHOP_ENTER、SHOP_ROLL
状态变化：phase='shop'；state.shop.offers 生成商品。
验收点：首次进入商店免费刷新；night_base 池可出商品。
对应测试：T58、T80

P10 商店刷新
玩家看到：点击刷新后商品列表重置；有免费刷新时不扣钱，没有免费刷新时扣1金币。
系统命令：ROLL_SHOP
系统事件：SHOP_ROLL、SHOP_ROLL_BLOCKED
状态变化：gold 扣费或 freeRolls 扣减；offers 更新。
验收点：免费刷新、付费刷新、金币不足阻止刷新都覆盖。
对应测试：T59、T60、T61、T96

P11 商品冻结/解冻
玩家看到：点击冻结后该商品保留到下次刷新；再次点击可解冻。
系统命令：FREEZE_OFFER、UNFREEZE_OFFER
系统事件：SHOP_FREEZE、SHOP_UNFREEZE
状态变化：offer.frozen true/false；刷新后冻结商品仍存在。
验收点：冻结、解冻、无效商品ID、冻结后刷新保留都覆盖。
对应测试：T62、T63、T80

P12 商店购买
玩家看到：点击购买商品后金币减少，商品进入背包；金币不足时提示不能买。
系统命令：BUY_OFFER
系统事件：SHOP_BUY、SHOP_BUY_BLOCKED
状态变化：goldFrom→goldTo；inventory 增加或同名合成；offers 移除该商品。
验收点：成功购买、金币不足、商品不存在、同名合成都覆盖。
对应测试：T64、T65、T66、T94

P13 元素/定位/品质商店池
玩家看到：火/水/风/土商店、定位商店、品质池商店能根据事件或池ID出对应商品。
系统命令：ENTER_SHOP / APPLY_SHOP_EVENT
系统事件：SHOP_ROLL、SHOP_EVENT_APPLY
状态变化：activePool/offer.poolId 变化；offers 出自对应池。
验收点：elem_火/水/风/土、role_输出/坦克/治疗/召唤/经济/控制/机动、tier_pT1/pT2/pT3 都可刷新。
对应测试：T68、T69、T70、T72、T78

P14 商店事件
玩家看到：夜晚商店出现事件，例如免费刷新、元素补货、定位补强；选择后扣金币或给收益。
系统命令：APPLY_SHOP_EVENT
系统事件：SHOP_EVENT_APPLY、SHOP_EVENT_BLOCKED
状态变化：gold、freeRolls、nextDiscount、offers 可能变化。
验收点：32条事件中的正式商店事件都能被尝试执行，且产生成功或阻止事件。
对应测试：T71、T72、T73、T74、T78

P15 离开商店
玩家看到：夜晚结束，显示剩余金币、背包宠物数量、遗物数量。
系统命令：EXIT_SHOP
系统事件：SHOP_EXIT
状态变化：phase='day_end'
验收点：完整日流程最后必须有 SHOP_EXIT。
对应测试：T77、T80、T95

对应测试：T91

对应测试：T92

五、当前明确边界
1. 当前是纯文字核心，不接 UI、不接动画、不接 DOM。
2. 玩家“点击格子”的真实坐标选择还未做成独立 Command；当前用自动目标选择模拟玩家选槽后的目标确认。
3. 机制系统已覆盖全部 ID 的识别和主要可计算钩子；部分复杂机制以事件流记录为第一版，不代表最终数值表现已经完全定稿。
4. V8 分支覆盖率不是100%，但行覆盖和函数覆盖是100%，业务验收链路按本文档全覆盖。
